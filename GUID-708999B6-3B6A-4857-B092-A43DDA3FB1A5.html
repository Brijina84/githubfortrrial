Decision Tables

A decision table allows you to easily define and maintain a large number of rules in a tabular format. Each rule in a decision table evaluates or sets the same object properties in the same order, but tests different values.

Decision tables are rule metaphors, where each table row represents a rule and every rule is evaluated. Decision tables can replace many of the rules that compare the same data fields in one or more rulesets but they are not intended for very large indexed lookup operations.

For example, you can create a decision table for credit-line assignment. Rules can evaluate income ranges and score ranges to determine the appropriate credit limit for each combination. The Decision Module provides functions that select the table or tables best suited for processing each step and application, that make the individual decision tables small and efficient.

Decision tables can be of three types based on the results they return:
  • Integer Table
  • Real Table
  • String Table

You can create and access decision tables from the Item Types tab of the navigation pane or navigate from a Decision Flow that uses the decision table.
