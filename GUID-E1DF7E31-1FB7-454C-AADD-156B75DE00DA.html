Message Summarization

When multiple Decision Module messages occur within the Application/MessageList for the same transaction, the highest message severity code returned is assigned to the entire transaction. This number appears in the StatusCode attribute of the MessageList element.

For example, if a level 2 warning and a level 3 error occur in the Application/MessageList, the overall transaction StatusCode will be 3, as shown in the following figure. 

<Application>
  <MessageList StatusCode="3" StatusDescription="System Error">
    <Message MessageNumber="10001" SeverityCode="2">
      <Description>Product Has Expired&lt;/Description>
      <Resolution>Select an active product and try
      again.</Resolution>
    </Message>
    <Message MessageNumber="11001" SeverityCode="3">
      <Description>Blaze Advisor System Error.
      {0}</Description>
      <Resolution>Contact <ph varref="FIName" 
      id="GUID-1A48373C-767D-4D65-B902-936C8B21B257"
      outputclass="variable">FICO</ph> Product
      Support.</Resolution>
    </Message>
  </MessageList>
</Application>

An Application/MessageList node with level 2 and level 3 errors

If no messages with a severity level of 1, 2, or 3 are returned, the overall transaction is assigned a StatusCode of ‘0’ and a StatusDescription of “Successful.” 

<MessageList StatusCode="0" StatusDescription="Successful"/>
