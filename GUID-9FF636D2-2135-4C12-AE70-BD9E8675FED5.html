Example: Adding a Time-based Wait Event

A time-based wait event is timer that you can add at a point where the workflow or process waits for a human action to be taken. Specify the processing flow if the activity is not performed within the interval of time specified.
In the following example, you add a time-based wait event to a User Activity event. Add a Get Data task and then add a human task to take some action. Link the human task to a Set Data task. Now add a time-based wait for the human task. If the user does not perform the required action within the specified time, you can either specify a different processing flow or connect the workflow back to the Set Data task.
  1. Drag and drop the Start event.
  2. Drag and drop the Get Data task, and then connect the Start event to the task.
  3. In the PROPERTIES pane, click ADD PROPERTY to add a parameter.
  4. Add a name for the parameter and click the Edit icon for the Value field to open the Select Path dialog box.
    The dialog box lists all the objects in the application BOM.
  5. Select the field in the BOM from which you want to retrieve the value and store in the parameter that you added in the previous step.
  6. Drag and drop the User Activity event on the canvas and connect the Get Data task to the Human task or user activity.
  7. Select the page to which the user would be taken to perform the task.
  8. Drag and drop the Set Data task and connect the User Activity to it.
  9. In the PROPERTIES pane, to select the object in the application BOM whose value you want to set, click the Edit icon next Path field.
  10. Select the data field in the application BOM.
  11. Specify the value either by specifying the value directly or by selecting the value from the drop-down list.
    In this case, the drop-down list will contain the outputs from the Get Data and the User Activity event.
  12. Drag and drop the End event , and then connect the Set Data task to it.
  13. To add a Time-based wait and configure it for the User activity or human task, perform the following steps:
    1. Drag and drop the Time-based Wait event.
    2. Select the User Activity from the Node drop-down list.
    3. Specify the interval of time for the user to perform the activity, failing which the timer is triggered.
    4. Specify the process flow if the activity is not performed and the timer is triggered.
    5. Connect the Time-based wait event to the Set Data task (which is the direction of the processing flow if the User activity is not performed.
  14. Click PUBLISH.
