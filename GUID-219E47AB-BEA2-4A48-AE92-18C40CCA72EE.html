Domain Model

Origination Manager provides out-of-the-box domain models for each line of business. You may edit the model to suit your business needs. A domain model consists of entities and fields. An entity corresponds to a table in a database, while a field corresponds to a column in that table.

An entity is the basic structure or building block of a domain model. Application is the root entity or element. You can add other entities under the root entity. You can add fields to an entity. The domain model contains certain core entities and fields that can neither be edited or deleted.
