Ruleset Severity Threshold

You have the option of determining whether the Decision Module should use a ruleset severity threshold when it processes the rules in a ruleset.

By default, a ruleset is set to return a candidate decision when any of its rules fires. To specify a ruleset severity threshold instead, you select If total rule severity exceeds from the second drop-down list for the Return decision setting, and then enter a numeric value for the threshold.

When a ruleset severity threshold is defined, the ruleset, after processing all rules, checks whether the cumulative severity level of its fired rules exceeds the threshold value. If yes, a candidate decision is registered. If not, the ruleset does not register a candidate decision.

The following example involves a ruleset for investigating whether applicant items are duplicates.


Ruleset severity threshold set at 599

The ruleset consists of rules with varying levels of severity:
  • Six low-risk rules, each with a severity level of 100
  • Three medium-risk rules, each with a severity level of 200
  • Two high-risk rules, each with a severity level of 300
  • One extreme-risk rule with a severity level of 600

Note that the figure shows only one rule for each severity level.

The Decision Module processes all of the above rules, and then calculates the cumulative severity level of the rules that fired. The ruleset severity threshold of 599 is exceeded when one of the following happens:
  • All six low-risk rules are fired
  • All three medium-risk rules are fired
  • Both high-risk rules are fired
  • The extreme-risk rule is fired
  • A combination of rules of different severities, such as four low-risk rules and one medium-risk rule, is fired

The ruleset then returns a candidate decision, along with the list of rules that fired.
