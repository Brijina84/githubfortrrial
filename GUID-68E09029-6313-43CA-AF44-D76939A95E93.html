Assigning an Expression

The Assign Expression dialog provides the ability to author action expressions. Action expressions or statements assign a value to a property as a consequence of a series of branch nodes (conditions) evaluating to true or false. Action expressions or statements are fired from top to bottom. If there is more than one action statement that assigns a value to a property, the value assignment overwrites any assignment that was previously made.
Note There is no undo command to revert the delete operation. However, you can click Cancel to discard unsaved changes.
The Assign Expression dialog has the following components:
  • The Expression label is the text that will display on the Action node in the product selection tree. This is a required field.
  • The Item Selector on the left hand side displays the entities associated with the category such as, product and subtrees.
  • On the right side, you can drag and drop the entities and change their order.
