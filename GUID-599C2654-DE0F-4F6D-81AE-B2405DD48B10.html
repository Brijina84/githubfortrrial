Adding a Condition Gateway

When you want the workflow to follow a processing sequence based on the evaluation of a condition, add a condition gateway, add condition values, and the processing sequence for each of the condition values being true.
Condition values to be evaluated are based on the available outputs of the previous step. For example, one of the available outputs of the previous step is a decision Decline application, you can specify one sequence of steps when Decline is true and another sequence of steps when Decline is false. By default, when you add a condition gateway, the designer adds a scenario that you needs to be configured if none of the conditions are true.
Note

In the following example, add a router gateway after the MakeDecision sub-process.

  1. Drag and drop the Start event.
  2. Drag and drop the MakeDecision sub-process, and connect the Start event to MakeDecision.
  3. Drag and drop a Condition gateway and connect it to MakeDecision.
    The Otherwise condition is added by default.
    Note For scenarios where none of the conditions are met, the Otherwise sequence is triggered.
  4. Click ADD CONDITION, and specify a name for the condition.
  5. From the list of available outputs, to select a condition and specify the condition value, click the expression icon.
    The EDIT EXPRESSION dialog opens. The MakeDecision subprocess has four available outputs, as seen in the image below:
  6. Click MakeDecision.Decision from the Available Outputs list.
    The output appears in the Output Property field.
  7. Select the operator for the condition and then type the value (Decline) for the condition or comparison. The operators allowed are Equals, Not Equals, Greater Than, Less Than, Greater Than Equal, and Less Than Equal.
    For example, select the operator Equals and type the Value True.
  8. Click DONE.
  9. If the condition equates to true, select what the next task from the Next Task dropdown list.
  10. To add more conditions, follow Steps 4-9.
  11. Add the next sequence for the Otherwise condition.
  12. Click PUBLISH to validate and publish the workflow or business process.
