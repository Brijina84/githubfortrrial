Data Method Context

The context of a data method is the full path of the object on which the data method is executed. The context path is displayed as a sequence of object names, separated by periods, with the first object always being the Application object.
When you build a data method, you can write an expression that calls other data methods that are in the scope of the original data method. The called data method can be defined for any object in the parent hierarchy of the calling data method. Consider the following example:
  • dm_Application has the Application object as its context.
  • dm_creditApplication has the Application.creditApplication object as its context.
  • dm_personalApplicant has the Application.creditApplication.personalApplicant object as its context.
  • dm_applicantScore has the Application.creditApplication.personalApplicant.applicantScore object as its context.
  • dm_scoreReason has the Application.creditApplication.personalApplicant.applicantScore.scoreReason object as its context.
  • dm_offer has the Application.creditApplication.offer object as its context.

In the example, dm_applicantScore can directly call dm_personalApplicant, dm_creditApplication, and dm_Application because the called data methods are defined on objects in the parent hierarchy of dm_applicantScore.

If you try to call dm_offer in dm_applicantScore, you will receive a validation error because dm_offer is not in the current scope of dm_applicantScore.

You can call a data method defined for a child object by iterating through the list of child objects using a for loop. In the example, you can call dm_scoreReason in dm_applicantScore by iterating through the available scoreReason objects as shown: 
for every Application.creditApplication.personalApplicant.applicantScore.scoreReason relative to applicantScore do 
dm_scoreReason;
endfor
Note If a data method is contained in a for loop within some other data method, the context within the loop can be different from that of the outer expression. So you can effectively access data at any location within the object model hierarchy, but must first establish context for the data or data method you plan to use. For example, consider a data object reference dor_CreditRequestData, which has the context Application.CreditRequest. In the expression for the data method dm_personalApplicant, you can have the following snippet: 
for every Application.CreditRequest in dor_CreditRequestData do
dm_creditApplication;
endfor
