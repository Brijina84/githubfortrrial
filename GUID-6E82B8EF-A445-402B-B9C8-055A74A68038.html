Uploading Custom Files

From the Custom Files and Functions option on the Tools tab, upload the custom files that contain the custom functions. Select the functions that you want to enable so that you can use them in data methods. You can uplaod one file at a time.
You must have SMW system administrator privileges to view and use the Custom Files and Functions option.
For more information about how to implement custom functions, see the FICO Origination Manager Decision Module Developer's Guide.
Note In FAC, the JAR files that you upload must be compiled in Java 1.7 compatible mode.
  1. In the Tools tab, select Custom Files and Functions from the left navigation pane.
  2. On the Custom Files and Functions page, click the Add icon.
  3. Browse to the location of the custom files, and select the file that you want to upload.
    Note The page lists the amount of occupied space in the repository. Ensure that the files you upload do not exceed the free space available in the repository.
  4. Click Upload.
    The Select Custom Functions to Add dialog box opens. The dialog box lists the custom classes in the file.
  5. Select the check box next to the custom classes or functions that you want to add, and then click Add.
    The selected classes are the enabled functions. You can use only the enabled functions in the data methods.
Note For on-premise, repackage the WAR files, to make the custom functions available for use in the data methods. For DMP, restart the component to make the custom functions available for use in the data methods.
