Reviewing and Processing Applications

When all data has been correctly entered for an application, it is submitted for processing according to the parameters set by your system administrator and the workflow processing steps of the system.

The application is first checked for data consistency and validation errors configured for your system. If any exist, the Application Processing Module generates an error message so you can correct the incorrect or missing data. If the key data matches data found in another application that was previously submitted, the system determines that the application is a possible duplicate (if your system is configured to check for duplicates) and places it in the appropriate queue until an analyst accesses it to make a determination. Likewise, the system evaluates the application for other issues (based on rules and strategies in the Decision Module, if it is used), such as missing data or possible fraud. When requested, the system can obtain a credit report for the applicants based on the product policy or the decision party of an application, if your company is using the Data Acquisition Module. Your system administrator will have previously set up accounts with each consumer or small business credit reporting agency (CRA) that your company wishes to use.

If your company is using the Analytic Module, the system can analyze application, applicant, and credit report information to calculate scores for consumer, and business applications using the FICOÂ® Application Risk Model (ARM), and the Small Business Scoring Service (SBSS) respectively. These scores can be compared to a score cutoff configured by your company.

The Application Processing Module evaluates the application and all data collected based on your company's configuration (for example, CRA reports). Depending on how your system is configured and the Origination Manager modules purchased, the system verifies and applies your policy rules, and either automatically accepts or declines the application, or sends the application to the queue you may have defined for pending reviews where a credit analyst can access and evaluate it, and make a decision.

Covered Topics
  • Viewing and Analyzing Application Data
    Use the Application page to view all of the data available for an existing application.
  • Adding Authorized Users
    In the Synchronous Consumer LOB, if the configuration permits, the UI displays the option to add authorized users and lets you add authorized users to a credit application An authorized user is a person who is authorized to use the credit card of the primary applicant. You can add authorized users to an application during application entry or while reviewing an application.
  • Adding Notes to an Application in Review
    If you use Synchronous Consumer Line of Business, use Notes to add information to an application during application review. Notes record information about an application. Notes can be free-form text, predefined notes, or any of the notes categories. For example, Applicant called to check status of application can be a pre-defined note that you add to an application.
  • Verifying Consumer Credit Applications
    When a consumer credit application contains information that needs to be verified, it is placed in Pending Verification status. Examples of such information are the applicant's monthly income or time with the current employer.
  • Verifying Synchronous Consumer Credit Applications
    When a synchronous consumer credit application contains information that needs to be verified, it is placed in Pending Verification status. Examples of such information are the applicant's monthly income or time with the current employer. The application verification step must be added to the synchronous workflow for processing applications.
  • Verifying Small Business Applications
    When a Small Business application contains information that needs to be verified, it is placed in Pending Verification status. Examples of such information are the net profit or the reported assets.
  • Obtaining CRA Reports
  • Resolving Application Issues
    If the Application Processing Module encounters an issue when processing a submitted application, the application is placed in the appropriate queue for resolution by an analyst.
  • Reviewing and Modifying Credit Offers and Entering Credit Offer Responses
    Access the Credit Offer Details pane of an application to review or edit a credit offer or to enter a customer or a business credit offer response.
