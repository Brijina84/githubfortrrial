Decision Levels

When a ruleset registers a candidate decision, the Decision Module sets the decision level to the value you specified in the Return decision setting.

The Return decision setting has two drop-down lists. You specify a value for the decision level using the first drop-down list. You select whether you want to define a ruleset severity threshold using the second drop-down list.

By default, the ruleset registers a candidate decision when any rule fires. If a ruleset severity threshold is defined instead, the ruleset registers a decision when the total value of rule severities produced by its fired rules exceeds the value of the threshold. In either case, when a decision is registered, the Decision Module sets the decision level to the value you specified.

The decision level can be one of the following values:
  • Review, Verify, User 1, User 2, User 3, Re-evaluate, Possible Fraud
  • Decline
  • Recommend Decline
  • Investigate
  • Recommend Approve
  • Approve
  • No Decision

In the first Return decision drop-down list, decision levels are listed in the order of precedence, from most conservative to least. The first seven decision levels are user-defined decision levels and have the same severity.

With the default value of No Decision, the ruleset returns a decision level of No Decision.
