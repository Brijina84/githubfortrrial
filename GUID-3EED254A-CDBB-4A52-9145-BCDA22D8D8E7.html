Working with Items

Covered Topics
  • Product Categories and Products
    Product categories are groups of related products. A product can be a loan, credit card, or other services provided by an institution. You can define multiple champion-challenger strategies for a product.
  • Managing Product Strategies
    A product strategy is a specialized decision tree that determines which decision flow executes for a product. The Decision Module derives a decision using a product strategy that is based on a organizations business rules and policies.
  • Decision Flows
    A decision flow is a graphical representation of processing associated with a Decision Module request.
  • Data Object References
    A data object reference (DOR) is a convenient and a user-friendly way to extract one or more objects that match specific conditions. Data object references increase performance and efficiency in processing applications.
  • Data Methods
    A data method is a procedure associated with a specific type of object within the FICO Origination Manager Decision Module domain model.
  • Data Method Sequences
    A data method sequence is an ordered list of data methods. Data method sequences are decision flow step implementations that execute the referenced data methods in the order that they are listed.
  • Rulesets
    A ruleset is a categorical grouping of rules and the parameters associated with each included rule. A ruleset is a group of related rules that execute on specified data. Rules in the same ruleset are evaluated at the same point within the decision flow step. From a strategy point of view, rulesets let you maintain sets of rules that work together to achieve a particular business decision.
  • Decision Tables
    A decision table allows you to easily define and maintain a large number of rules in a tabular format. You can create and access decision tables from the Item Types tab of the navigation pane or navigate from a Decision Flow that implements the decision table
  • Decision Trees
    A decision tree is a graphical representation of a ruleset. Use a decision tree to create and maintain rules in a tree format.
  • Score Models
    A score model is a mathematical formula used to predict the likely behavior of a customer or prospect.
  • Scenarios
    A scenario is a specialized type of decision tree action that represents possible outcomes.
  • Score Model Reason Code List
    In addition to returning a score, you can configure a scorecard to return a reason code for each bin that is fired. Reason codes provide users with reasons as to why a particular score was assigned during evaluation.
