Managing Users

Administrators with the proper permissions use the Application Processing Module user management pages to create and modify users and assign roles and groups to them.
Note OM identifies a user by the email address or user name. A user needs to log on to APM with the email address or user name and password.

You can specify whether APM user login must happen by using either an email address or a username. The site property, UseEmailAsUsername can be set to true or false. When set to true, user login must be done using an email address; when set to false, user login must be done using user name. This configuration is a one-time activity and when set, applies to all clients. Use the FASt Manager to set the site property in the om_apm_users eSpace.

Covered Topics
  • Creating Users
    Administrators with the proper permissions can create new users on the Users > Create User page.
  • Maintaining Users
    Administrators with the proper permissions can search for users and update user profiles.
