Data Object References

A data object reference (DOR) is a convenient and user-friendly way to extract one or more objects that match specific conditions. Data object references increase performance and efficiency in processing applications. Create data object references once and reuse them as needed.

Use data object references like objects in strategy components such as rules, data methods, decision tables and decision trees. Variables in strategy components like decision trees and decision tables can be assigned data object references to extract objects for further processing.

Preconfigured data object references use a naming convention that begins with "dor_".

Access data object references from the Item Types tab. Once you create a data object reference, define the data object reference from the Data Object Reference Properties page.

When you define a data object reference, you must select a context in the object model for the data object reference. Use the object model browser to select a context for the data object reference. A data object reference returns an object in the context.

You can also define input arguments for a data object reference. Use input arguments to create a generic data object reference that can accept an input value and evaluate conditions based on the input. Data object references with inputs can be used only by data methods. Add conditions to evaluate the fields in the data object reference context or in the hierarchical context of the parent object.
