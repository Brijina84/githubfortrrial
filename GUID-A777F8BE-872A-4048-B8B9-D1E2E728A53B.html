Decision Module Messages

The following table shows Decision Module warning and error messages, along with descriptions and possible resolutions. Each error or warning message is listed by the number that appears in the Application/MessageList/Message/@MessageNumber output attribute.
Decision Module Error and Warning Messages
10000    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Product Is Not Active
  Resolution Select an active product (or make the product active by changing its effective start date) and resubmit the request.
10001    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Product Has Expired
  Resolution Select an active product (or make the product active by changing its effective end date) and resubmit the request.
10002    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Decisioning Request Type ({0}) Is Not Supported

(where {0} is the Decisioning Request Type)

  Resolution

Ensure that the value submitted for Application/@DecisioningRequestType is compatible with the rules used for decisioning and resubmit the request.

10003    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Product Category ({0}) Does Not Exist

(where {0} is the Product Category)

  Resolution

Ensure that the value submitted for CreditRequest/@ProductCategory is a valid Product Category in the rules used for decisioning and resubmit the request.

10006    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Product Category Is Blank Or Missing
  Resolution

Ensure that a valid value is submitted for CreditRequest/@ProductCategory and resubmit the request. The value must match a Product Category set up in the rules used for decisioning.

10007    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Product Code Is Blank Or Missing
  Resolution

Ensure that a valid value is submitted for CreditRequest/@ProductCode and resubmit the request. The value must match a Product Code set up in the rules used for decisioning.

10008    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Decisioning Request Type Is Blank Or Missing
  Resolution

Ensure that a valid value is submitted for Application/@DecisioningRequestType and resubmit the request. See the description of DecisioningRequestType in the COS data model specification for more information.

10009    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Product ({0}) is Invalid for Product Category ({1})

(where {0} is the Product Code and {1} is the Product Category)

  Resolution

Use a valid combination of Application/CreditRequest/@ProductCode and Application/CreditRequest/@ProductCategory and resubmit the request.

10010    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Unable to execute Data Object Reference: {0}.
  Resolution The configuration within a Data Object Reference caused an error when processing. Please check the configuration and try again.
10011    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Internal Service ({0}) Error: {1}
  Resolution Analyze the error text to resolve the problem. Ensure the internal service is properly configured, using the Internal Services screen in the Strategy Management Workstation. Also ensure the Blaze Advisor project associated with the internal service is properly deployed, and XML transform files, if used, are valid.
10012    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Web Service ({0}) Error: {1}
  Resolution Analyze the error text to resolve the problem. Ensure the web service is properly configured, using the Web Services screen in the Strategy Management Workstation. Also ensure the web service is properly deployed and accessible to Decision Module and that XML transform files, if used, are valid.
10013    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Product Code Not Available From Product Selection Tree {0}
  Resolution

Ensure that a valid value is submitted for CreditRequest/@ProductCode or configure the Product Selection Tree to return a Product Code and resubmit the request. The value must match a Product Code set up in the rules used for decisioning.

10014    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description No Decision Flow defined for {0}/{1} (where {0} is the ProductCategory and {1} is the ProductCode)
  Resolution

Ensure that the product definition has a valid Decision Flow for the product specified by CreditRequest/@ProductCode within the category specified by CreditRequest/@ProductCategory and resubmit the request. Each product requires a default Decision Flow or a Decision Flow to be returned by a Product Strategy in order to make decisions.

10020    
  Product Name Decision Module
  Severity Code 3 (System Error)
  Description Invalid Callout in Decision Flow
  Resolution Check the configuration of the decision flow and all components that it references.
10021    
  Product Name Decision Module
  Severity Code 3 (System Error)
  Description Unknown Action in Decision Flow
  Resolution Check the configuration of the decision flow and all components that it references.
10030    
  Product Name Decision Module
  Severity Code 2 (User Error)
  Description Invalid Data In Schema
  Resolution Ensure that the data in the schema is valid and resubmit the request.
11001    
  Product Name Decision Module
  Severity Code 3 (System Error)
  Description Blaze Advisor System Error
  Resolution Contact FICO Product Support.
15000    
  Product Name Decision Module
  Severity Code 3 (System Error)
  Description {0}
  Resolution {1}

This is a runtime error from the Data Method functions, used inside data methods. {0} is the detail about the exception. {1} is additional trace information.
