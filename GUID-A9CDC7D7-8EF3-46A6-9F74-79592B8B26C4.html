Reason Rank Order

Each rule is associated with a reason, and the Decision Module returns the reason associated with each rule that fires to the calling application. For each candidate decision, the Decision Module includes a list of reasons that contributed to that decision, and a special Rule History entry containing a list of reasons for rules that fired but whose ruleset did not register a candidate decision.

Since it is possible that many reason codes will be returned for each candidate decision, rank ordering is set so that the reasons for the rules that are deemed to be the most significant are displayed higher in the list than reasons that may be of less importance. Note that the system cannot resolve a tie; if two reasons are given the same rank and both are returned, there is no way to determine which reason is shown first.

When you configure a rule, you determine whether a rule’s reason and severity will be registered if the rule fires. If you specify a decision reason for a rule, the rule’s reason and severity level are registered if the rule fires.
