Scoring Scenarios

A scoring scenario is a mechanism for recording decisions based on scores returned by score models executed by the scenario. Each scoring scenario is defined as one or more score model information rows; each row specifies a score model, its upper and lower cutoff ranges, and the decision result to return if the score returned by the score model falls within the specified cutoff range. Scoring scenarios are selected within an action node of a decision tree.

Each score model is capable of operating on any data available at the point in processing where the scoring scenario was executed. When a scoring scenario is executed, it creates a Decision object that is returned to the calling application. If desired, other Decision object fields can be populated by executing a data method sequence in the same decision tree action node, or in a later decision flow step.

Covered Topics
  • Scoring Scenario Properties
    The Scoring Scenario Properties page displays the properties associated with a scoring scenario, such as the maximum and minimum scores and the decision associated with a score model.
  • Creating a Scoring Scenario
    Use the Create button on the Scenarios list page to add a new scoring scenario.
  • Copying a Scoring Scenario
    Use the Save As icon in the Scoring Scenario Properties page to add a new scoring scenario based on an existing one.
  • Deleting a Scoring Scenario
    Use the Delete icon on the Scenarios list page to delete a scoring scenario that is no longer needed.
  • Deleting a Score Model Row
    Use the Remove current score condition icon in the Score Model Information section to remove a score model row in a scoring scenario.
