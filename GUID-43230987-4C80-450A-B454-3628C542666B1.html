Example: Champion-Challenger Decision Flows

You can use preconfigured mappings to implement champion-challenger decision flows.
Consider a product strategy where you can use DecisionRequestType and StrategySelectionRandom Number mappings to create champion and challenger paths. In this product strategy, 10% of the pre-bureau requests (those with random numbers from 0-9) will be processed using the challenger decision flow and the remaining 90% (those with random numbers from 10-99) will be processed using the champion decision flow. Post-bureau requests follow the path to a post-bureau decision flow.
Note The StrategySelectionRandomNumber field generates random numbers between 0-99.
The following steps describe how to create the champion and challenger paths:
  1. Configure a condition node and select the DecisionRequestType in the Type field.

  2. Add two conditional branches, Pre-bureau and Post-bureau.

  3. In the Pre-bureau branch, configure a condition node using the preconfigured StrategySelectionRandomNumber type, and add two transition conditions. The first sends 10% of the requests (those with random numbers from 0-9) down one branch and the second sends all others (those with random numbers from 10-99) down the second branch.

  4. Then create two action nodes. In the first one, select the DS2_PreBureau decision flow. In the second, select the DS1_PreBureau decision flow.

  5. Finally, select the DF3_PostBureau decision flow for the Post-bureau branch action node.

After you create the champion-challenger decision flows, you can easily modify an existing challenger, add a new challenger, or promote an existing challenger to champion by changing assigned percentages. This allows you to fine-tune your lending strategies by easing into a new strategy, changing small things over time, and reviewing the resulting outcomes. Champion-challenger decision flow implementation is a powerful method of responding quickly to market trends with small modifications to your system.
