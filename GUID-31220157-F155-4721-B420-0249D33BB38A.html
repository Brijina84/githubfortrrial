Example: Adding an Assignment Event

An assignment event assigns or sets a value to a variable or field in the application BOM. You can use an assignment event in both, sub-process and process or workflow.
In the following example, create a sub-process with a few input, output, and local variables. Add a Get Data task which fetches data from the application BOM, and then add an assignment event. You can also assign the output of the Get Data Task to the variables. Then create a process and add the sub-process that you have created.
  1. Drag and drop the Start event.
  2. Drag and drop the Get Data task, and then connect the Start event to the Get Data task.
  3. In the PROPERTIES pane, click ADD PROPERTY to add a parameter.
  4. Add a name for the parameter and click the Edit icon for the Value field to open the Select Path dialog box.
    The dialog box lists all the objects in the application BOM.
  5. Select the field in the BOM from which you want to retrieve the value and store in the parameter that you added in the previous step.
  6. Click MANAGE PARAMETERS to open the Manage Parameters page.
  7. Add input, output, and local parameters for the sub-process.
  8. Select Assignment from the ELEMENTs list and drag it onto the canvas, and then connect the Get Data task to it.
  9. Assign values to parameters in the ASSIGNMENT PROPERTIES pane. To assign a value to a parameter, perform the following steps:
    1. From the Assign list, select the parameter to which the value is assigned.
    2. From the To list, select the parameter whose value is assigned.
      You can specify any additional operation on the value being assigned to the parameter or variable.
    3. To add another parameter for assignment, click ADD PROPERTY and repeat Steps a and b.
  10. Click PUBLISH.
  11. Create a process.
  12. Drag and drop the Start event.
