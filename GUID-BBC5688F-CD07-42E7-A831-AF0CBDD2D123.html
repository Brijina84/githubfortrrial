DSS Update Queries

In addition to updating the final decision, you must use DSS update queries to update the analyst and override codes for the Override Tracking Report and update the final offer for the Pricing Decision Analysis Report. 
UPDATE decisionflowhistory1 AS target
    INNER JOIN (
    SELECT tran.hncid, dfh.hncid AS dfhid, superid
    FROM datransaction_1 AS tran
        inner join decisionflowhistory_1 AS dfh ON tran.hncid =
        dfh.superid
    WHERE tran.SystemId = 'sysid'
        AND tran.ApplicationCrossReferenceId = 'APPREF'
        AND dfh.ProductCode = 'PRODUCT'
        AND dfh.DecisionFlowName = 'DECISONFLOW'
        AND dfh.ProductCategory = 'CATEGORY'
    ORDER BY ProcessingTimestamp DESC
    LIMIT 1
    ) AS source ON source.dfhid = target.hncid
      AND source.superid = target.superid
    SET FinalDecisionOperator = 'XXXXXX';
UPDATE decisionflowhistory1 AS target
    INNER JOIN (
    SELECT tran.hncid, dfh.hncid AS dfhid, superid
    FROM datransaction_1 AS tran
        inner join decisionflowhistory_1 AS dfh ON tran.hncid =
        dfh.superid
    WHERE tran.SystemId = 'sysid'
        AND tran.ApplicationCrossReferenceId = 'APPREF'
        AND dfh.ProductCode = 'PRODUCT'
        AND dfh.DecisionFlowName = 'DECISONFLOW'
        AND dfh.ProductCategory = 'CATEGORY'
    ORDER BY ProcessingTimestamp DESC
    LIMIT 1
    ) AS source ON source.dfhid = target.hncid
      AND source.superid = target.superid
    SET FinalDecisionOverrideCode = 'XXXXXX';
When using DSS update queries, keep the following in mind: 
insert into scoremodelbinbaseline values(<score model name>,<characteristic name>,<bin label>,<baseline>)
Where:
  • <score model name> must be the same as the score models that are created. Used by the Characteristic Analysis, Population Stability, and System Decision by Score Range Reports.
  • <characteristic name> must be the same as the name of the characteristic used in the corresponding score model. Used by the Characteristic Analysis and Population Stability Reports.
  • <bin label> must be the same as the bin label of the corresponding score model. Used by the Characteristic Analysis Report.
  • <baseline> is used by the Characteristic Analysis Report. 
insert into scoremodeldevbaseline values(<score model name>, <range lower>, <range upper>, <baseline>)
Where:
  • <score model name> must be the same as the score models that are created. Used by the System Decision by Score Range Report.
  • <range lower> is the inclusive lower range. It does not need to be the same as the lower range of the score model. Used by the System Decision by Score Range Report.
  • <range upper> is the inclusive upper range. It does not need to be the same as the upper range of the score model. Used by the System Decision by Score Range Report.
  • <baseline> is used by the Population Stability Report.
