Function Calls

Data method expressions can contain function calls—operations that invoke a function. A function is a named procedure in a data method that performs a specific task.

There are over 40 functions defined in the SMW. These functions are described in detail in Data Method Expression Language.

A function has the following syntax: 
FunctionName(function_inputs)

where function_inputs is a comma-separated list of inputs.

Inputs can include any inputs (of the correct data type) that are defined for the specific function. If there are no values or references defined for the function, the parentheses are still used, but contain nothing: 
FunctionName()

The following is an example of a data method that contains a function call.

Function Call
Item Value
Name dmw_Product_CreateOffer
Description Create the Offer Object
Context Application.DecisionResponse.Product
Result Type numeric
Local Variable 
Name   Data Type   Initial Value
Offer  xOffer      null
Expression result := null; 
xOffer := New Offer;
Attach(Product,xOffer)
The expression reads as follows:
  1. Assign result an initial value of null.
  2. Create a new object whose object type is Offer, and assign it to the xOffer local variable.
  3. Using the Attach function, attach the new instance of Offer that has been created to the Product object.
