Adding a Router Gateway

When you want the workflow to follow a processing sequence based on the evaluation of a condition, add a condition gateway, add condition values, and the processing sequence for each of the condition values being true. The condition values are based on field values in the OM application BOM.
In the following example, add a router gateway after the ExperianUSConsumer sub-process. The ExperianUSConsumer sub-process returns some data which is stored in the application BOM.
Note With a router gateway, you can add conditions on only one field value in the BOM.
  1. Drag and drop the Start event.
  2. Drag and drop the ExperianUSConsumer sub-process, and connect the Start event to ExperianUSConsumer.
  3. Drag and drop a Router gateway and connect it to ExperianUSConsumer.
    The Otherwise condition is added by default.
    Note For scenarios where none of the conditions are met, the Otherwise sequence is triggered.
  4. To select the object in the application BOM on which the condition is to be applied, click the Edit icon next to the OM Application Path field.
  5. Select the data field in the application BOM, which contains a value returned by the ExperianUSConsumer and you want to add conditions on that value.
  6. Click SELECT.
    You can now add conditions on the value in the field referenced by its XPath, as seen in the OM Application Path.
  7. Click ADD CONDITION and specify a name for the condition.
  8. Select the operator for the condition and then type the value for the condition or comparison. The operators allowed are Equals, Not Equals, Greater Than, Less Than, Greater Than Equal, and Less Than Equal.
  9. Select what the next task from the Next Task dropdown list.
  10. To add more conditions, follow Steps 7-9.
  11. Add the next sequence for the Otherwise condition.
  12. Click PUBLISH to validate and publish the workflow or business process.
