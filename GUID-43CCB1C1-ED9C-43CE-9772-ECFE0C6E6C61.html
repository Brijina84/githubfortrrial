Verification Error Scenarios

The following table describes some of the common error scenarios that can occur during verification of strategy components and corrective actions to resolve them.
Error Messages, Descriptions, and Resolutions
Data method <name> not found
  Item Data method sequence
  Description

The data method sequence row references a non-existent data method. A data method sequence contains rows specifying the data method to be invoked and the object(s) upon which it will be invoked. If you rename or remove the data method or remove its context object from the object model, the reference becomes invalid. Importing data may also introduce invalid data methods.

  Resolution

Open the Data Method Sequence Properties page, locate the problem row, select the row, select a valid data method from the Object Model Browser, and save.

Data method <name> not found
  Item Decision table
  Description

The decision table contains a reference to a variable that is mapped to a non-existent data method. A decision table uses variables to define the data used in condition nodes. These variables, configured in the Manage Variables section, must reference a valid field or data method. If you rename or delete the field or data method used in the variable, the reference becomes invalid.

  Resolution

Open the Decision Table Properties page. Locate the problem variable, assign it to a valid data method or field, and save.

Data method <name> not found
  Item Decision tree
  Description

The decision tree contains a reference to a variable that is mapped to a non-existent data method. A decision tree uses variables to define the data used in condition nodes. Each variable must be assigned to a valid field or data method. If you rename or delete a data method after a variable is defined, the reference becomes invalid.

  Resolution

Open the Decision Tree Properties page. Locate the problem variable, assign it to a valid data method or field, and save.

Data method <name> not found
  Item Product selection tree
  Description

The product selection tree contains a reference to a variable that is mapped to a non-existent data method. A product selection tree uses variables to define the data used in condition nodes. if a variable is mapped to a data method, the data method must be valid. If you rename or delete a data method mapped to the variable, the reference becomes invalid.

  Resolution

Open the Product Selection Tree Properties page. Locate the problem variable, assign the variable to a valid data method or field, and save.

Data method <name> not found
  Item Product strategy
  Description

The product strategy contains a reference to a variable that is mapped to a non-existent data method. A product strategy uses variables to define the data used in condition nodes. Each variable can be mapped to a field or data method. If you rename or delete a data method after a variable is defined, the reference becomes invalid.

  Resolution

Open the Product Strategy Properties page. Locate the problem variable, assign the variable to a valid data method or field, and save.

Data method <name> not found
  Item Score model
  Description

The score model characteristic contains a reference to a variable that is mapped to a non-existent data method. A score model characteristic uses a variable to define the data it uses. Each variable can be assigned to a field or data method. If you rename or delete a data method after a variable is defined the reference becomes invalid.

  Resolution

Open the Score Model Properties page. Locate the problem variable, assign the mapping to a different data method or field, and save.

Data method <name> not found
  Item Rule
  Description

The rule condition “operand 1” contains a reference to a data method that does not exist. Each rule condition includes two operands, each of which must be a valid field or data method. If you delete or rename a data method after the rule condition has been defined, the reference becomes invalid.

  Resolution

Open the Ruleset Editor, find the rule, and open it. Find all conditions in the rule that reference the data method specified in the verification error, specify a different data method or field, and save; or create a data method at an appropriate context to make the reference valid and save.

Data method sequence <name> not found
  Item Decision tree
  Description

The decision tree contains a Select Data Method Sequence action node that specifies a non-existent data method sequence. A decision tree Select Data Method Sequence action node is used when a data method sequence is executed from a decision tree. If you later rename or delete the data method sequence, the action node reference becomes invalid. You can also introduce an invalid data method sequence reference when importing a decision tree.

  Resolution

Open the Decision Tree Properties page for the invalid decision tree, open the Node Properties for the action node, select a valid data method sequence, and save.

Data object reference <name> not found
  Item Data method sequence
  Description

The data method sequence row specifies a non-existent data object reference. When configuring a data method, you can specify a data object reference to control the objects on which the data method will be executed. If you rename or delete the data object reference, the reference becomes invalid. Importing data may also introduce invalid data object references.

  Resolution

Open the Data Method Sequence Properties page, locate the problem data object reference, select the row, select a valid data object reference or object from the data object browser, and save.

Data object reference <name> not found
  Item Decision table
  Description

The decision table contains a reference to a variable that is mapped to a field or data method using a data object reference that does not exist. A decision table uses variables to define the data used in condition nodes. Each variable can be assigned to a field or data method, and for each variable you can optionally specify a data object reference to determine the object to which the field or data method is associated. If you rename or delete a data object reference used in the variable, the reference becomes invalid.

  Resolution

Open the Decision Table Properties page, and specify a valid data object reference or object, and save.

Data object reference <name> not found
  Item Decision tree
  Description

The decision tree contains a reference to a variable that is mapped to a field or data method using a data object reference that does not exist. A decision tree uses variables to define the data used in condition nodes. Each variable can be assigned to a field or data method, and for each variable the user can optionally specify a data object reference to be used to determine the object to which the field or data method is associated. If you rename or delete a data object reference after the variable is defined, the reference becomes invalid.

  Resolution

Open the Decision Tree Properties page, assign a valid data object reference or object, and save.

Data object reference <name> not found
  Item Product selection tree
  Description

The product selection tree contains a reference to a variable that is mapped to a field or data method using a data object reference that does not exist. A product selection tree uses variables to define the data used in condition nodes.   Resolution

Data object reference <name> not found
  Item Product strategy
  Description

The product strategy contains a reference to a variable that is mapped to a field or data method using a data object reference that does not exist. A product strategy uses variables to define the data used in condition nodes. Each variable can be assigned to a field or data method, and for each variable you can optionally specify a data object reference to determine the object to be associated with the field or data method. If you rename or delete a data object reference after the variable is defined, the reference becomes invalid.

  Resolution

Open the Product Strategy Properties page, specify a valid data object reference or object, and save.

Data object reference <name> not found
  Item Rule
  Description

The rule filter specifies a non-existent data object reference. For each rule, you can optionally specify a data object reference filter to limit the evaluation of conditions to only the object(s) at the rule context that satisfies the constraints of the specified data object reference. If you delete or rename the data object reference after the rule condition is defined, the reference becomes invalid.

  Resolution

Open the Ruleset Editor, find and open the rule, select a valid data object reference, and save; or re-create or rename an existing data object reference to match the name specified for the rule.

Data object reference <name> not found
  Item Score model
  Description

The score model characteristic contains a reference to a variable that is mapped to a field or data method using a data object reference that does not exist. A score model characteristic uses a variable to define the data it uses. Each variable can be mapped to a field or data method, and for each variable you can optionally specify a data object reference to determine the object to which the field or data method is associated. If you rename or delete a data object reference after the mapping is defined, the reference becomes invalid.

  Resolution

Open the Score Model Properties page, specify a valid data object reference or object, and save.

Decision flow <name> does not exist
  Item Product strategy
  Description

The product strategy action node specifies a non-existent decision flow. A product strategy action node is used to select a decision flow. If you rename or delete a decision flow after the product strategy is defined, the reference becomes invalid.

  Resolution

Open the Product Strategy Properties page and then open Node Properties for the action node, select a valid decision flow, and save.

Decision flow <name> not found
  Item Product
  Description

The product references a non-existent decision flow as the default decision flow. A default decision flow is executed if a product strategy is not defined, or if one is defined, but a decision flow is not actually selected during execution. The product's default decision flow name must match the name of an existing decision flow. If you rename or delete the default decision flow, the reference becomes invalid.

  Resolution

Open the Product Properties page, select a valid default decision flow, and save.

Decision table <name> not found
  Item Decision tree
  Description

The decision tree contains a Select Table action node that specifies a non-existent decision table. A decision tree Select Table action node is used when a decision table is executed from a decision tree. If you rename or delete the decision table, the action node reference becomes invalid. You can also introduce an invalid decision table reference when importing a decision tree.

  Resolution

Open the Decision Tree Properties page and then open the Node Properties for the action node, select a valid decision table, and save.

Decision tree <name> not found
  Item Decision tree
  Description

The decision tree contains a Select Tree action node that specifies a non-existent decision tree. A decision tree Select Tree action node is used when a decision tree is executed from a decision tree. If you rename or delete the decision tree, the action node reference becomes invalid.   Resolution

Open the Decision Tree Properties page and then open the Node Properties for the action node, select a valid decision tree, and save.

Decision flow Step Result <value> not found
  Item Decision tree
  Description

The decision tree contains a Select Result action node that specifies a result value that is not defined for this decision tree. A decision tree Select Result action node is used when the action node needs to set the decision tree result value. The result value is returned by the decision tree to control branching in the calling decision flow. If you later rename the specified decision tree step result value or remove it, the action node's result value becomes invalid.

  Resolution

Open the Decision Tree Properties page and then open the Node Properties for the action node, select a valid step result value, and save.

Invalid data method expression: <message> ()
  Item Data method expression
  Description

Errors presented in this format indicate that a data method expression is syntactically incorrect. Incorrect expressions may be caused by errors during data entry or importing of new data. For example: “Invalid data method expression: Expecting a numeric expression in field 2 [“Abc”]”.

  Resolution

Open the data method editor and find the problem data method. Click the Validate Calculation button in the Data Object Browser, identify the source of the error, correct it, re-validate, and save.

Variable <name> not defined
  Item Decision table
  Description

The decision table contains a reference to a variable that is not mapped. A decision table uses variables to define the data used in condition nodes. Each variable used in the decision table must have a field or data method assigned to it.

  Resolution

Open the Decision Table Properties page, assign the variable to a valid field or data method, and save.

Product <name> not found
  Item Product
  Description

The product references a non-existent product as its upsell or downsell product. A product may specify an upsell or downsell product to be evaluated if an Approve or Decline decision is returned for the current request. The upsell or downsell product name must match the name of an existing product within the current product category. If you rename or delete a product, or move a product to another category, the reference becomes invalid.

  Resolution Open the Product Properties page, select a valid upsell or downsell product, and save.
Product <name> not found
  Item Product selection tree
  Description

The product selection tree action node specifies a non-existent product. A product selection tree Select A Product action must specify a valid product name. If you rename, delete, or move a product after the product selection tree is defined, the reference becomes invalid.

  Resolution

Open the Product Properties page and then open Node Properties for the action node, select a valid product, and save.

Product code is not defined
  Item Product
  Description

The product code is not defined. A product code is a required field used at runtime to identify a product. If you remove it or leave it blank, the product will not be executed.

  Resolution

Open the Product Properties page, change the product code to a valid value, and save.

Product has been defined without either a default decision flow or product strategy
  Item Product
  Description

The product has been defined without a default decision flow or product strategy; one or the other is required. When a request is processed for a specific product, processing is controlled by a product strategy, or if one is not defined, a default decision flow. If neither is defined, the product will do nothing.

  Resolution

Open the Product Properties page, select a product strategy, a default decision flow, or both, and save.

Product selection tree <name> not found
  Item Category
  Description

The category references a non-existent default product selection tree. The name of the default product selection tree must match a product selection tree defined for this category. If you rename a product selection tree after it has been assigned as the default, the reference becomes invalid.

  Resolution

Open the Category Properties page, change the default product selection tree to a valid value, and save.

Product selection tree <name> not found
  Item Product selection tree
  Description

The product selection tree action node specifies a non-existent sub-tree. A product selection tree Select Sub-Tree action is used to select another product selection tree to execute. The product selection tree name must be valid. If you rename or delete a product selection tree that is referenced by another product selection tree the reference becomes invalid.

  Resolution

Open Node Properties for the action node, select a valid product selection tree, and save.

Product strategy <name> not found
  Item Product
  Description

The product references a non-existent product strategy. A product strategy is executed when a request is to be evaluated for the product. The product strategy name must match the name of an existing product strategy. If you rename a product strategy, the reference becomes invalid.

  Resolution

Open the Product Properties page, select a valid product strategy, and save.

Rule specifies a ruleset step result that is not defined for this ruleset
  Item Rule
  Description

The rule specifies a ruleset step result that is not defined for this ruleset. For each rule, you can optionally specify a ruleset step result value to be returned by the ruleset. It is used to control branching in the decision flow that invokes the ruleset. If you remove it from the ruleset, the reference becomes invalid.

  Resolution

Open the Ruleset Editor and then find and open the rule. Select a valid ruleset step result value, and save; or modify the ruleset's list of valid ruleset step result values to include the value specified for this rule.

A decision tree Select Ruleset action node is used when a ruleset is to be executed from a decision tree. If you rename or delete a ruleset that is used in a decision tree or decision table, the action node reference becomes invalid. You can also introduce an invalid ruleset reference when importing a decision tree.

Score model <name> not found
  Item Scoring scenario
 

The scoring scenario score model information row references a non-existent score model. In each scoring scenario score model information row, you specify a score model to be executed. If you rename or remove a score model, the reference becomes invalid.

  Resolution

Open the Scoring Scenario Properties page, find the score model information row, select a valid score model, and save; or open the score model page and rename the score model or create a new one with that name and save.

Server Config File <name> not found
  Item Internal service
  Description

The internal service references an invalid server configuration file. For each internal service, you must specify a server configuration file. If you rename or remove the file after the internal service is defined, or if you promote the repository to a different environment without copying the server configuration file, the reference becomes invalid. An invalid server configuration file name can also be introduced during an import.

  Resolution

Open the Internal Services Properties page, select a valid server configuration file from the list, and save.

Service name <name> is not found in Server Config File
  Item Internal service
 

The internal service references an invalid service name for the selected server configuration file. For each internal service, you must specify a rule service to be invoked when the Decision Module executes the internal service. The rule service must be defined in the server configuration file specified for this internal service. If you rename or remove the rule service or the server configuration file after the internal service is defined, or if you promote the repository to a different environment without copying the server configuration file, the reference becomes invalid. An invalid rule service name can also be introduced during an import.

  Resolution

Open the Internal Services Properties page, select a rule service name from the service name list, and save.

Table result not defined (Decision Table)
  Item Decision table
  Description

The decision table's result is not mapped to a field in the object model. The result must map to a valid field in the object model.

  Resolution

Open the Decision Table Properties page, map the result to a valid field in the object model, and save.
