Object-based Processing Loops

An object-based processing loop is used to execute a statement sequence on all instances of a specific context path, relative to the active context at the point in the expression where the loop is inserted. This type of loop is most commonly used when the data you need is in an object that is a child of the current context. For example, if your context is applicantFinancial, which has a child object named income, you can use an object-based processing loop to traverse through all income objects associated with the applicantFinancial object.
Object-based processing loops must conform to the syntax shown in the following statement: 
for every <context_path> relative to <object_type> do <statement_sequence> endfor
Where:
  • <context_path> is the current active context. For more information, see Active Context in Loop Statements.
  • <object_type> is the parent object in the active context. For example, if Application.creditApplication is the active context, <object_type> is Application.
  • <statement_sequence> is a simple or complex statement, conditional statement, object-based processing loop, or data object reference-based processing loop that is executed once for each object in the context specified by <context_path>.
  • If there are multiple statements within statement_sequence, each statement prior to the last must end with a semicolon. The comma is optional for the last statement.

The following data method is an example of an object-based processing loop.

Object-based Processing Loop
Item Value
Name dm_EvaluateApplicantFinancial
Description Calculates the total grossMonthly income for each applicant.
Context Application
Result Type numeric
Local Variable 
Name                    Data Type              Initial Value
xGrossMonthlyIncome     numeric                0
Expression result := 0; 
for every Application.creditApplication.personalApplicant.applicantFinancial.income relative to applicantFinancial do
xGrossMonthlyIncome := xGrossMonthlyIncome + grossMonthlyIncome;
endfor
applicantFinancial.totalGrossMonthlyIncome := xGrossMonthlyIncome;
result := xGrossMonthlyIncome;
This expression reads as follows:
  1. Assign the result an initial value of 0.
  2. For every income object whose parent is a applicantFinancial object (whose parent in the Application object), assign to result the value of result plus the value of grossMonthlyIncome.
  3. Stop when all such income objects have been evaluated.
The following keywords are used in this statement:
  • for every and endfor show the beginning and end of the statement and direct the system to repeat the operation, subject to any constraints that follow.
  • relative to constrains the loop to only evaluate income objects whose parent is a applicantFinancial object whose parent is the Application object.
  • do directs the system to execute the next statement sequence.
