Score Models That Return Distanced Reasons

Choose this option when you want to return reason codes ranked according to the distance of the generated characteristic score from the characteristic baseline. The baseline score for each characteristic is a user-specified value that modelers usually provide and is considered to be a neutral value for score purposes. The model returns score reasons only for characteristics that score below the baseline. When distanced reasons are returned, you see the topmost 10 reasons listed in descending order, from highest to lowest distance.

For example, suppose a safe driver rating is based in part on the number of accidents a driver has had. The following table shows how the Decision Module computes score reason distances.

Number of (At-fault) Accidents Characteristic
Number of Accidents Score Unexpected Flag Score Reason Rank
None at fault 50 X 01 5
0 35   03 3
1 20   02 1
2 or more 10   02 1
If the baseline score for this characteristic is 30 and Number of Accidents value is 1, the distance of the score (20) from the baseline of 30 would be 10 or: 
score reason distance = baseline - score

When you choose the Score Model Distanced Reasons option, priority-order ranking is used for tie-breaking purposes where there is more than one computed score reason distance that is the same.

Note Consult with your legal counsel to ensure that any user-specified baseline score comports with legal requirements.
