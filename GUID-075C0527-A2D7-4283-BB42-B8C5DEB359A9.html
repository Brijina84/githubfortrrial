Example: Adding an Error Handler Event

An error handler catches an error in a task, call-activity, or sub-process and specifies how the processing should flow when the error occurs. You can add multiple error handlers in a workflow. An error handler must be connected to the error code and the construct or the node that throws the error. An error handler catches specified errors or all errors, and specifies the error handling logic and the processing flow when these errors occur.
In the following example, you add an error handler for the call-activity, REST service. Assume that the REST service throws three errors OM001, OM002, and OM003. Add an error handler for two of the errors, OM001 and OM002, and then add an event (human activity) to take the processing flow in another direction if OM001 or OM002 occur.
Note The same error code might be thrown by more than one construct. To connect an error handler to a particular error, you must specify the error code and the construct or the node that throws the error.
Note A construct or node can throw more than one error. For example, a call to a REST service can give multiple errors. In such cases, you can specify a comma-separate list of error codes and specify the processing flow when those errors occur.
  1. Drag and drop the Start event.
  2. Drag and drop the REST service call-activity, and than connect the Start event to the call-activity.
  3. Drag and drop the End event, and then connect the sub-process to it.
  4. Drag and drop the Error Handler event.
  5. In the Properties pane for the Error Handler, enter OM 001, OM002 in the Error Code field and select the REST service call-activity from the Node drop-down list.
  6. Drag and drop a User Activity,and connect the Error Handler event to it.
  7. Configure the User Activity event by selecting the page that must open or load for the user activity.
  8. Click PUBLISH.
