Using Filter Functionality for Queues

Set the filter criteria of the queue definition. The queue will contain applications meeting these filter criteria. For example, you can create a queue containing applications that have a status of Pending Credit Offer Entry for a specific product that were submitted by a certain date.
  1. Click Add condition.
  2. Select the first field to filter.
    The drop-down list of fields contains both the contents of the Available and Selected lists.
  3. Select the operator. The available operators depend on the type of data associated with the selected field.

    For numeric or date fields, logical operators such as is equal to, is greater than, and is less than appear. For string fields, string operators such as is empty and is not empty to appear.

  4. (Optional) If it is enabled, select the check box in the Other Field column to populate the Value column with domain values instead of a field in which you enter information.
    Note
    • The Other Field check box is not enabled for all fields. For example, it is enabled for fields such as Applicant First Name because this is a text type field. It is not enabled for fields such as Address State because this field contains domain values.
    • If you choose one of the date fields as filter criteria, a calendar is displayed for you to select a date.
  5. (Optional) Click Add condition to add a new line to the filter.
    A new row appears below the previous line of the filter. Use this row to specify how the line will be used with the new one. For example, select AND if both the result of the first line of the filter and the result of the second line of the filter must be true for the record to display.
  6. Choose from the following operators from the new drop-down list that appears at the beginning of the new line of the filter. Use this list to specify how the previous filter will be used with the new one:
    • Select AND if both the result of the first line of the filter and the result of the second line of the filter must be true for the record to appear.
    • Select OR if either the result of the first line of the filter or the result of the second line of the filter must be true for the record to appear.
  7. (Optional) Use the Group conditions feature to refine the search filter.
    For example, (Application Status is equal to 'Pending Credit Offer Entry' AND Application Product is equal to 'Product ABC') OR Submitted On is greater than or equal to '2013-08-30'. Use the Ungroup conditions feature to remove the search filter refinement.
  8. Continue with the next steps needed to create or modify the queue.
