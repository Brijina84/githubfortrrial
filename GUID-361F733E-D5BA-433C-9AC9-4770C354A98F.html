ExcelCellValue

Syntax: ExcelCellValue(workbookName, worksheetName, columnHeaderValue, columnComparisonType, rowHeaderValue, rowComparisonType)

Description: Reads an Excel file and returns data to the calling data method.

Inputs:

  • workbookName: The Excel file.
    Note Only letters, numbers, spaces, and underscores are allowed in file names.
  • worksheetName: The worksheet that contains the lookup table.
  • columnHeaderValue: The value to search for in column headers.
  • columnComparisonType: The column value type—Alpha, Date, Numeric.
  • rowHeaderValue: The value to search for in row headers.
  • rowComparisonType: The row value type—Alpha, Date, Numeric.
The function supports six operators:
  • =
  • <
  • >
  • <=
  • >=
  • <>
You must specify the operators for the row and column in the Excel file as shown in the following example:


Row and Column Operators Specified in the Worksheet

Return Type: string

Returns: Returns the Excel cell value that matches the specified criteria. To check for errors, use this function along with ExcelCellValueIsError and ExcelCellValueLastError.

Example: 

xMsgList:= new MessageList;
xLoanOfficer:= newloanOfficer;
xLoanOfficer.lendingLimit:= Val(ExcelCellValue("Book1","Worksheet1","Amount1","Alpha","User3","Alpha"));

if(ExcelCellValueIsError("")="true" ) then
 xExcelError:= ExcelCellValueLastError("");
 xMessage:= new Message;
 xMessage.Description := xExcelError;
 Attach(xMsgList,xMessage);
endif

Attach(Application,xLoanOfficer);
Note The function always returns a string regardless of the data type specified in the Excel cell. You need to convert the returned string to the required type. In the example, the ExcelCellValue function is enclosed within the Val function to convert the returned string to a numeric representation before the value is assigned to lendingLimit.
