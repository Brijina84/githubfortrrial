Decision Flows and Their Components

A decision flow defines the decision performed for an application that requests a product. After you create a decision flow, access the decision flow from the Item Types tab.

The decision flow specifies the execution order of its components and the conditions under which to execute them. In the SMW, decision flows are represented graphically as a logical flow of processing steps. Each decision flow step performs a specific unit of work within the decision flow, and if necessary, makes a determination of the next step.

A decision flow step has a corresponding implementation type, which can be a specific ruleset, data method sequence, decision tree, decision table, or score model. Set transition conditions to specify what happens after a step is performed.

Using transition conditions, the first encountered path is followed. You can determine path direction and conditional branching by configuring ruleset or decision tree result values that are used by decision flow step transitions. You can create multiple steps that transition to a single step, or logical loops that repeat processes until a specified condition is met.

Decision flows are created using the following components and their subcomponents:
  • Rulesets
  • Rules (subcomponent)
  • Data Methods (subcomponent)
  • Data Method Sequences
  • Data Object References (subcomponent)
  • Decision Trees
  • Scenarios (subcomponent)
  • Score Models
  • Decision Tables
Covered Topics
  • Rulesets
    A ruleset is a categorical grouping of rules and the parameters associated with each included rule.
  • Rules
    A rule is the combination of one or many conditions and a resulting outcome that implements organizational decisions or policies.
  • Data Methods
    A data method is a procedure associated with a specific type of object within the domain model.
  • Data Method Sequences
    A data method sequence is an ordered list of data methods governed by user-defined properties.
  • Data Object References
    A data object reference (DOR) is a convenient and user-friendly way to extract one or more objects that match specific conditions. Data object references increase performance and efficiency in processing applications. Create data object references once and reuse them as needed.
  • Decision Trees
    A decision tree is a graphical representation of a ruleset. Use decision trees to create and maintain rules in a tree format.
  • Scenarios
    Scenarios are specialized actions that are configured within a decision tree to represent possible outcomes.
  • Score Models
    A score model is a mathematical formula used to predict the likely behavior of a customer or prospect.
  • Decision Tables
    A decision table allows you to easily define and maintain a large number of rules in a tabular format. Each rule in a decision table evaluates or sets the same object properties in the same order, but tests different values.
