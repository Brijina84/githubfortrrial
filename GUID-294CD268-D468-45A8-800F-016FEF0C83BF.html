Personal Data Acquisition Request Applicant Option Type Values

These values are used by the Data Acquisition Module adapter.

Personal Data Acquisition Request Applicant Option Type Values
Option Name Description
GENDER
(Optional) The gender of the applicant. May be required for compliance purposes.
  • 1 or M = MALE
  • 2 or F = FEMALE
  • 3 = NOT_APPLICABLE
  • 4 = NOT_SPECIFIED
REQUEST_ANALYSIS_FLAG

(Optional) Specifies whether consumer credit report analysis characteristics should be returned for the applicant. Y = Return analysis characteristics. N = Do not return analysis characteristics (default).

REQUEST_PRINT_IMAGE_FLAG

(Optional) Specifies whether a print image of the credit report should be returned for the applicant. Y = Return a print image. N = Do not return a print image (default).

REQUEST_RAW_ARF_FLAG

(Optional) Specifies whether raw ARF data should be returned for the applicant. Y = Request raw ARF data. N = Do not return raw ARF data (default). Raw ARF data is not returned if raw ARF data is submitted. If you submit raw ARF data on input, setting this flag has no effect; raw ARF data will not be returned on output.
