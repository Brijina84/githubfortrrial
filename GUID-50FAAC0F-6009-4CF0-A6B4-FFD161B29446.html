Decision Trees

A decision tree is a graphical representation of a ruleset. Use a decision tree to create and maintain rules in a tree format.

Creating a decision tree based on a business need is a multi-step process. You can create the parent node and the path structure, then return to configure the nodes. Create and configure nodes and branches until all the conditions are evaluated and return decisions.

Apart from setting a unique name and a description, you can set the following properties for a decision tree:
  • Decision Tree Results—List of result values that can be set in the action nodes.
  • Manage Variables—Variables that map to data fields and data methods in the object model.
  • Node Properties—Properties that define the behavior of a condition or action node.

A decision tree is built using condition nodes and action nodes. Condition nodes evaluate variables and branch the decision tree. Action nodes can return a decision, or a result that can be the step result in a decision flow. Action nodes can also call other item types that return results.

Use the editor in the Decision Tree Properties page to build the decision tree by configuring the condition nodes and the action nodes.

Covered Topics
  • Example: Score and Income Decision Tree
    A decision tree can return decisions based on an applicants FICO® score, debt-to-income ratio, and income amount.
  • Creating a Decision Tree
    Use the Create button in the Decision Trees list page to create a decision tree.
  • Importing a Decision Tree by using an FSML File
    Create a decision tree from a FICO Strategy Markup Language (FSML) file by importing the file. You can import FSML or XML files.
  • Decision Tree Tasks
    You can perform tasks, such as insert, copy, edit, clear, assign work on a selected tree node.
  • Inserting Splits
    Add a split to a decision tree when you want to create a set of branches for condition nodes.
  • Inserting Levels
    All condition nodes on a single level represent a segment of the same decision variable. When you create a decision tree, it does not contain any levels. Add a level using the Insert Level command in the Edit drop-down menu. You can also insert a level at the same time you insert a split using the Insert Split command.
  • Copying a Decision Tree
    Use the Save As icon in the Decision Tree Properties page to create a decision tree from an existing decision tree.
  • Copying and Pasting Subtrees
    A split with two or more branches is known as a subtree. You can copy a subtree from a given condition node and paste it to another condition node. You usually copy and paste subtrees when you want to repeat the same series of conditions on another branch using the same levels.
  • Reordering Branches
    Reorder the branches to rearrange their display order in the tree.
  • Creating Conditions Using Or by Merging Branches
    Merge the branches to combine condition expressions that are connected with "or". For example, to author an expression such as thePatient.serviceType is "Emergency" or thePatient.serviceType is "Office_Visit", merge the branches. You can only merge branches in levels of enumeration or string type.
  • Assigning an Expression
    The Assign Expression dialog provides the ability to author action expressions. Action expressions or statements assign a value to a property as a consequence of a series of branch nodes (conditions) evaluating to true or false. Action expressions or statements are fired from top to bottom. If there is more than one action statement that assigns a value to a property, the value assignment overwrites any assignment that was previously made.
  • Creating and Mapping Variables
    Create and map variables to data fields in the object model. Variables determine the fields that condition nodes evaluate when the tree is run.
  • Deleting a Decision Tree
    Use the Delete icon in the Decision Trees list page to delete a decision tree that is not required.
  • Decision Tree Profiling
    Use decision tree profiling to see how data flows through the decision tree logic and how the actions are distributed for a given sample dataset.
  • Exporting a Decision Tree
    Export a decision tree as a FICO Strategy Markup Language (FSML) 2.5 file to exchange tree models with other FICO products.
