Adding and Configuring a Rule Condition

Use the Add icon in the Rule definition section of the Rule Properties page to add and configure rule conditions.

The rule must be checked out and in edit mode before you can add conditions.

  1. In the Rule definition section of the Rule Properties page, click the Add a new condition icon.
    A row for specifying a condition is displayed.
  2. In the Field or Data method cell in the new row, add an item using one of these options:
    • If you know the name of the field or data method (or a part of the name), start entering it. From the drop-down list, select a value. For example, if the rule's context is Application.creditApplication.personalApplicant, typing A will bring up a list of fields and data methods that contain "A" in their names and are associated with any of these objects: Application, creditApplication, or personalApplicant.
    • Click the Open object model browser icon, select an item from the Object Model Browser, and click Select. If you do not know where in the object model the field or data method is defined, use the Search feature to locate it.

    If you have a data method that returns an object, it is possible to select a field associated with the object. For example, if you have a data method dm_applicant that returns the personalApplication object, you can select dm_applicant:personalApplicant.age in the left side of your condition expression. If you know a part of the name, you can type it in and select the field from the list displayed. Alternatively, you can select the data method from the Object Model Browser, and then select the field from the list displayed.

  3. Select a conditional operator from the Operator list.
  4. In the Field / Data method / Enumeration / Value cell, specify a value using one of these options:
    • If you know the name of the field or data method (or a part of the name), start entering it. From the list, select a value.
    • Click the Open object model browser icon, select a field or data method from the Object Model Browser, and click Select. If you do not know where in the object model the field or data method is defined, use Search to locate it.
    • Enter a valid enumeration (specified in the domain model) or a value. For string values, use double quotes to enclose the value. For dates, use the "yyyy-mm-dd" format.

      If you selected in or not in in the Operator cell, use a comma-separated list to add multiple values.

    Note You can select fields of the format DataMethodName:ReturnObject.ObjectField only in the left side of the condition expression, not in this cell.
    Note Comma is a special character. With the "IN" and "NOT IN" operators, do not use a comma separator inside a string.
  5. Click Save.

After you have added and configured all the rules, you may want to verify the associated ruleset (by clicking the Verify icon) to make sure that the rule definitions are valid.
