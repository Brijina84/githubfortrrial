Versioning Workspace Items

The SMW controls user access and implements versioning by enabling users to check in and check out items. Versioning allows multiple users to safely work on items without overwriting another person’s work.

Each SMW user has a private workspace that is connected to the configuration repository. When a new item is added, it is created within the user's private workspace. The item is added to the configuration repository and gets versioned when it is checked in. At any given time, only one user can check out and edit an item. Each time an item is checked out and then checked back in, a new version is created and saved.

For more information on working with items in your workspace, see Private Workspaces.

Item list pages display the following columns with version-related information:
  • Status—displays icons representing the edit status of the item.
  • Version—displays the current version number of the item, and an icon for viewing version history.
  • Owner—displays the user ID of the person who has checked out the item.
  • Last Modified—displays the date and time the item was last changed.

In the actions toolbar, Check In, Check Out, and Cancel Check Out icons are available when those actions are allowed.

Covered Topics
  • Checking Out an Item
    To edit an item in your private workspace, check it out from the configuration repository.
  • Canceling a Check Out Action
    When you have an item checked out, you can undo the Check Out action by canceling it.
  • Checking In an Item
    When you have completed working on an item within your private workspace, check in the item to commit your changes to the configuration repository.
  • Updating the Workspace
    To get the item changes that other users of the SMW have checked in, update your private workspace. Updating synchronizes your private workspace with the configuration repository.
  • Viewing Version History
    You can examine all versions of an item using the History option.
  • Promoting an Item Version
    The working copy of an item is the version that is currently active and editable when checked out. You can promote an earlier version of the item to be the working copy.
  • Version Comparison
    On an item's properties page, the History option in the Version drop-down list displays two tables—one containing information about the working copy, and the other listing all previous versions of the item. In the Version History table, you can select any two versions of the item and compare them.
