Gateways

Gateways control the flow of execution in a business process. Use a gateway to control the divergence of convergence of multiple sequence flows.
You can use the following types of gateways:
  • Parallel. Use this gateway when you want to bring about concurrency in a process.
  • Condition. Use this gateway when you want to bring about a decision in the process.
  • Router. Use this gateway when you want to bring about a decision, but applicable only on the OM application business object model (BOM).

Parallel gateway

Condition gateway

A condition gateway moves the workflow sequence along a path based on conditions. When a workflow execution reaches a condition gateway, all outgoing sequence flows are evaluated. When a condition evaluates to TRUE, that outgoing sequence is selected. Multiple sequences can be selected and the processing can proceed similar to a parallel flow. A router gateway is similar to a condition gateway, where you can specify the condition based on field values in the OM application BOM. Covered Topics
  • Adding a Condition Gateway
    When you want the workflow to follow a processing sequence based on the evaluation of a condition, add a condition gateway, add condition values, and the processing sequence for each of the condition values being true.
  • Adding a Router Gateway
    When you want the workflow to follow a processing sequence based on the evaluation of a condition, add a condition gateway, add condition values, and the processing sequence for each of the condition values being true. The condition values are based on field values in the OM application BOM.
