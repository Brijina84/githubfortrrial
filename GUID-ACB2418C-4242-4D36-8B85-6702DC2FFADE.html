Field Properties

A field is a property of an entity. It is similar to a

Add an entity by clicking the Add Entity symbol.

Note You can add a field only to an entity.
A field has the following properties:
Field Properties 
Name
 
Description
 
Type
 
Mandatory
Name Name for the field. String Yes
Description Description to help understand why the field is used. String Yes
Display name Name displayed for the field in the UI. Yes
Context Path to the entity from the root entity. - Auto-populated
Data type Data type of the data that the field will contain. Available types: string, boolean, integer, number, date, date time.
Note Based on the data type you select, the workbench displays the fields you must configure.
Multiple occurrence Whether the field has a one-to-many relations Boolean No
Mandatory Whether the field is mandatory. Boolean
Email Whether the field will contain an email. Boolean
Length Allowed length of the field.
Pattern RegEX pattern. This pattern will be used for validation during application submission (wherever this field can be updated).
Masked Whether the field is masked. The field is masked in the UI and Service. Boolean
Only alphabet and space Whether the field can contain only alphabets and spaces Boolean
Required for decisioning Whether the field is used in decisioning of the application. Boolean
Searchable Whether the field is searchable. Boolean
Secured Whether field contents are encrypted. Boolean
