Creating a Decision Scenario

Use the Create button on the Scenarios list page to add a new decision scenario.
  1. In the navigation pane, click Scenarios.
  2. On the Scenarios list page, click the Create button.
  3. In the Create dialog box, enter a unique name.
    Names must be text strings and can contain alphabets, numbers, and underscores.
  4. In the Description field, enter text describing the decision scenario.
  5. In the Type drop-down list, select Decision Scenario.
  6. Click Create.
    The new decision scenario is listed on the Scenarios list page.

    To specify the properties for the decision scenario, you need to edit the scenario. At this point, you can proceed with the following steps to set the properties, or you can check in the decision scenario to commit it to the configuration repository. If you check in the scenario, you need to check it out when you are ready to edit the properties.

  7. Click the decision scenario name on the Scenarios list page.
    The Decision Scenario Properties page appears.
  8. Click the Edit icon.
  9. In the Decision code field, enter text specifying the decision code for the calling system.
  10. From the Decision type list, select a decision level.
  11. In the Letter ID field, enter text specifying the ID for the calling system.
    Letters can be generated in the calling system based on the Letter ID provided by Decision Module.
  12. In the Queue field, enter text specifying the queue for the calling system.
  13. To add a reason to return with the decision response, click the Add a new reason icon in the Reasons section and specify the reason.
  14. Click Save.
