Test Execution

Test components and strategies in the SMW with different test scenarios. You can retrieve a base application and add use cases to test.

Use the Test Execution tab to test different use cases with a base application. Create use cases, test use cases, and review test results against actual results. Optionally, save the use cases as test scenarios in a private workspace in the SMW application server.

When you create test scenarios, add test attributes for the input test data that map to data fields in the object model using the object model browser. You can define the expected output test data for each use case. Add test attributes for the output test data using the object model browser.

You can also enter test attributes as XPath expressions. The SMW can suggest test attributes from data fields in the object model as you enter a field name. The input test attributes must conform to valid XPath expression formats. For more information see, XPath Expressions for Input Test Attributes.
Note The SMW does not validate the XPath expressions.

When you retrieve a base application and the relevant test scenario, the SMW displays the input test data and the expected output test data in a grid format. You can add or modify test attributes. Retest different use cases in real-time.

The SMW displays the following outcomes:
  • Actual result and expected result are identical.
  • Actual result and expected result are different. The SMW highlights results that vary from the expected output.
  • The test scenario failed to run due to a input or processing error. The SMW displays a link so that you may review errors in the response XML.

By default, the SMW shows all test results. For another example on test results, see also Example: Application Processing for Revolving Category

You can also access the Base Applications tab and the Test Scenarios tab from the Test Execution tab.
