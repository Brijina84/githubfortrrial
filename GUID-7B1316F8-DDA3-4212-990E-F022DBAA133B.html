Data Methods

A data method is a procedure associated with a specific type of object within the FICO Origination Manager Decision Module domain model. It performs one or more operations when executed, and may produce new information or take an action such as setting a field. You can use data methods to compute, manipulate, set, select, or return values using standard mathematical operations.

Data methods cannot be called directly from a decision flow, but are always invoked as data method sequences within steps. They can be grouped into data method sequences; used in rules, or as inputs for decision tables, decision trees and score models; or nested within other data methods. Data methods can also be used to execute internal services. Data methods are versatile and powerful tools that help you leverage other Strategy Management Workstation features.

To write or edit data methods, you need to:
  • Become familiar with the structure of the domain model, so you know what objects and fields are available and appropriate for a specific use.
  • Understand the types, structure, and components of a data method.
  • Learn to use the Data Method Expression Language to create syntactically valid expression statements.

    It is recommended that you use consistent naming conventions to facilitate data method identification.
  • Data Method Context
    The context of a data method is the full path of the object on which the data method is executed.

    Before building a data method, you typically gather relevant information and create an outline of the process.
  • Testing Data Methods: Overview
    By changing a data method expression's conditional logic and editing the XML input against which the data method is run, you can test various scenarios for multiple data sets directly from the Strategy Management Workstation.
  • Managing Data Methods
    Use the options on the list and properties pages for data methods to create, edit, validate, test, copy, and delete data methods.
  • Using the Data Method Expression Editor
    The Expression Editor is a code editor used to construct a data method expression that is evaluated when the data method is executed.
  • Using the Data Method Expression Language
    The Data Method Expression Language is a proprietary language designed for creating data method expressions. It allows the business analyst to implement complex data methods for use in strategies.
