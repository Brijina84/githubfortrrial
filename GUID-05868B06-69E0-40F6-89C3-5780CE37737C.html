Data Object References

Decision flow components such as data methods, data object references, rules, and mappings need a context, or specified set of objects, on which to operate. You specify a context by choosing the desired object from the Data Object Browser in the SMW.

Sometimes a decisioning request may contain multiple instances of one and the same objectâ€”for example, a previousResidence object. The domain model schema permits multiple instances of this type of object to be present. You may or may not want a decisioning rule to take all of them into account.

A data object reference is useful in this situation because it enables you to specify the instances of objects to use for decisioning. A data object reference returns a list of objects that satisfy a set of conditions; if multiple instances match the conditions, it returns multiple objects.

For example, if you create a data object reference to look at previous addresses, and the application input contains multiple previous addresses, the data object reference returns all the instances. You can use the data object reference as the context of a rule to cause the rule to operate on all the instances.

The Data Object Browser opens in the SMW. It displays a domain model in a tree structure. Child elements are shown indented a level from their parents. A data object reference appears as a leaf of its associated object.
