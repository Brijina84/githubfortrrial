Test Scenarios

Test scenarios define the different use cases to test components and strategies. You can save the test scenarios in the private workspace in the SMW application server and retrieve them to retest components and strategies.

Use the Test Scenarios tab to manage test scenarios.

Test scenarios include test data that contain input test data and output test data. When you create test scenarios, add input test data for different test cases and the expected output for each test case.

The test data grid contains the following fields :
  • Test Attribute. Data fields in the object model. You can add test attributes using the object model browser for input and expected output data. You can also enter test attributes as XPath expressions. The input test attributes must conform to valid XPath expression formats. For more information see, XPath Expressions for Input Test Attributes.
    Note The SMW does not validate the XPath expressions.
  • Case. Test cases define the values for test attributes in the input test data and output test data.

The SMW can also suggest test attributes from data fields in the object model as you enter a field name.

You can perform the following actions from the Test Scenarios list page:
  • Create a test scenario
  • Search for a test scenario stored in the repository by name or description
  • Create a copy of a test scenario
  • Delete a test scenario
  • Sort the test scenarios
