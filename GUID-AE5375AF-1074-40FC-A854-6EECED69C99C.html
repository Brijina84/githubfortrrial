Scenarios

A scenario is a specialized type of decision tree action that represents possible outcomes.
There are three types of scenarios:
  • Decision scenarios—Produce decisions and associated reasons.
  • Pricing scenarios—Produce product offerings for an applicant.
  • Scoring scenarios—Execute score models and produce decisions based on scores returned from those score models.
Each type of scenario is available for selection when configuring the action node of a decision tree.
Note Best practice dictates that a decision tree would typically contain action nodes with all scenarios of the same type. So a decision tree that is used as a scoring strategy, for example, would contain only scoring scenarios.

Scenarios can be reused across multiple categories and products, and are accessed from the navigation pane.

Covered Topics
  • Decision Scenarios
    A decision scenario is a mechanism for recording decisions from within an action node of a decision tree.
  • Pricing Scenarios
    A pricing scenario is a mechanism for recording product offers from within an action node of a decision tree. Each pricing scenario specifies parameters for one or more product offers, including offer ID, interest rate, credit limit, and term.
  • Scoring Scenarios
    A scoring scenario is a mechanism for recording decisions based on scores returned by score models executed by the scenario.
