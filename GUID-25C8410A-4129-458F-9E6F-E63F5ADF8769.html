Uploading Images

Before anyone can use an image in a document template, the image must be uploaded to the Application Processing Module and stored.
After an image is uploaded, it appears in the table on the Document Templates > Images page.
Note You can insert or update images in existing document templates created with Origination Manager Application Processing Module 4.6 or earlier releases. When you create new document templates, the Word document you browse and select from your filesystem must include the images you want to display.
The Document Templates > Images page includes a table that lists all of the images in the system. For each image, the table shows the name of the image, a preview of the image itself, the dimensions of the image, and the date the image was uploaded or modified. Note that the image name is the name of the file you upload to the system, for example, CompanyLogo.jpg. You cannot modify the name unless you replace the image using a new image name.
The maximum size, height, and width of the image are defined by site properties in the om_apm_docgen_core eSpace. You can modify these settings by changing the default values for the site properties in FICO Application Studio Manager.
Note From the Origination Manager APM 5.0 release onwards, SVG and CGM file types are no longer supported.
  1. In the administration navigation pane, click Document Templates > Images.
  2. Click Upload.
  3. Enter the path to where the image currently resides, or click Browse to browse to its location.
  4. Click Upload. Preview the image of the file. Click Discard to enter a different image or click Accept to upload the image to the system.
    A row will be added to the image table with the following information: the image, image name, dimensions, and modified date.
