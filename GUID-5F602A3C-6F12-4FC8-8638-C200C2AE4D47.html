Rulesets

A ruleset is a categorical grouping of rules and the parameters associated with each included rule. A ruleset is a group of related rules that execute on specified data. Rules in the same ruleset are evaluated at the same point within the decision flow step. From a strategy point of view, rulesets let you maintain sets of rules that work together to achieve a particular business decision. For example, a Minimum Requirements ruleset would contain rules that, together, ensure the application meets the minimum credit policy requirements established for a product. Products may have multiple rulesets. The Decision Module reads the application data and uses the rules defined in the ruleset to evaluate the application and return a decision.
Grouping your rules can improve performance by focussing on a specific group of rules and specific attributes in the domain model. From an execution point of view, rulesets behave like independent "islands" of business logic because:
  • Evaluation proceeds independently of other rulesets.
  • The Decision Module evaluates data using the currently active ruleset before proceeding with the next ruleset in the decision flow.

The resulting decision that a ruleset returns depends on the decision level. By default, the ruleset registers a decision when any rule fires. When ruleset severity threshold is enabled, after the ruleset completes processing all rules, if the cumulative severity level for the ruleset exceeds the ruleset severity threshold, the ruleset registers a decision. When you configure a rule, you can optionally indicate a candidate ruleset step result to register if the rule fires.

Covered Topics
  • Decision Levels
    When a ruleset registers a candidate decision, the Decision Module sets the decision level to the value you specified in the Return decision setting.
  • Ruleset Severity Threshold
    You have the option of determining whether the Decision Module should use a ruleset severity threshold when it processes the rules in a ruleset.
  • Ruleset Results
    Rules are capable of setting result values that can be used in a decision flow to determine the next decision flow step.
  • Managing Rulesets
    Use the options on the ruleset list and properties pages to create, edit, copy, and delete rulesets for lending products.
  • Rules
    A rule is a combination of one or more conditions with a resulting decision reason and severity level that represent your organization’s decisions and policies.
