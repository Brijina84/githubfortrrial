Creating a Data Object Reference

Use the Create button in the Decision Object References list page to create a data object reference(DOR).
  1. In the Data Object References list page, click the Create button.
    The Create dialog box appears.
  2. Enter a unique name for the DOR in the Name field .
    Names must be text strings of 150 characters or less. Names can contain letters, numbers, and underscores.
  3. Optionally, enter a description in the Description field.
  4. Click Create.
    The new DOR appears in the Data Object References list page.
  5. Click the DOR to view the Data Object References Properties page.
  6. Optionally, click the Edit icon to modify properties.
  7. Click Change to change the object context for the data object reference.
    Select a context in the object model using the object model browser.
  8. Click the Add icon in Input arguments section to define input arguments.
    The system assigns a default name and data type for the input argument. You can modify the the name and data type of a input argument.
    1. Click the default name.
      The Data object reference argument properties page appears.
    2. Modify the Name of the input argument.
    3. Modify the Type of the input argument from the drop-down list.
    4. Click Save.
    5. Click the DOR name in the breadcrumb trail at the top of the Properties page to return to the Data Object References Properties page.
      Repeat the steps to add multiple input arguments. You can add or delete input arguments, however, you must edit any conditions in the data object reference where the input argument is used as well as any data methods that use the data object reference. You must not change the order of the input arguments once a DOR is defined.
  9. Add conditions for data object reference to extract objects that match certain criteria. You can configure the data object reference to return objects when all conditions are true or when any condition is true.
    Build condition expressions by selecting the fields, operators and values.
    1. Click Add to add a condition expression row.
    2. Click Open the object model browser to select a Field.
      You can select a field in the data object reference context or the hierarchical context of the parent object.
    3. Select a valid Operator from the drop-down list.
      Pre-defined operators appear in the list based on the data type of the Field.
    4. Select a Enumeration or enter a value that can be a input argument or a user-defined value.
    Repeat the steps to add multiple conditions that the data object reference can evaluate and return objects that match the conditions.
  10. Click Save.
