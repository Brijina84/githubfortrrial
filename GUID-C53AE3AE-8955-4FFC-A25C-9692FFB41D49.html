Decision Tree Profiling

Use decision tree profiling to see how data flows through the decision tree logic and how the actions are distributed for a given sample dataset.

You can use profiling for your decision trees. When the profiler runs, the condition nodes in your decision tree evaluate each record in a dataset. The data that matches the conditions and actions is calculated and displayed in the Decision Tree editor as a record count.



Decision Tree Displaying in Tree View

Covered Topics
  • Dataset Requirements for Decision Tree Profiling
    The dataset used for decision tree profiling must be a CSV (comma-separated values) file that conforms to specific requirements. You can create your own dataset in an external application or generate a sample dataset and modify the values.
  • Generating a Sample Dataset for Decision Tree Profiling
    Generate a sample dataset based on the current decision tree. The generated .csv shows the correct format for the dataset and includes a column header and column for each condition in a decision tree. After the sample dataset generates, you can modify the values as required.
  • Previewing and Running Data Profiling
    To run the decision tree profiler, attach a .csv dataset to run against the conditions in the decision tree. Only one dataset can be attached to a decision tree.
