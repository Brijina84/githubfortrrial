Private Workspaces

The Strategy Management Workstation has a configuration repository that stores the items used to implement business strategies. Each SMW user has a private workspace that is connected to the configuration repository. Users can create, edit, and delete items within their private workspaces. Changes made within a private workspace are saved to the configuration repository when the items are checked in. After this, other users can see the changes when they update their private workspaces; they can check out and edit the items provided they have the appropriate permissions.

You can find the location of your private workspace in your Session Information window. If you are logging on to the SMW for the first time, or your private workspace is empty when you log on, the configuration repository is copied into your private workspace.

Having a private workspace allows you to control when to make your changes available to others and when changes made by other users are visible in your private workspace. You will see the changes made by other users when you update your private workspace and synchronize it with the configuration repository.



Private Workspaces

In the diagram you see two users, each with a private workspace. Note that the contents of the workspaces are different. User 1 has created Item B in Private Workspace 1. Item B will get versioned and be available for use by User 2 only when User 1 checks it into the configuration repository. When User 2 subsequently updates Private Workspace 2, Item B, as well as the changes in Version 3 of Item A, will be visible.

The Decision Module requires items to have unique names; however items with identical names can be created accidentally. Consider a scenario when User 1 creates an Item C (a unique name) and checks it into the repository. Then User 2 also creates an Item C (same name) and then updates the private workspace. In the private workspace of User 2, the item type's list page lists two items with the same name. Validation of unique item names occurs only when User 2 checks in Item C into the repository.

Note Before you delete or rename an item, perform a Where Used search to determine where the item is used. If there are references, remove them before you proceed. After you have made your changes, use the Verify feature to make sure that your actions did not result in errors.
