Scoring Scenario Properties

The Scoring Scenario Properties page displays the properties associated with a scoring scenario, such as the maximum and minimum scores and the decision associated with a score model.

The Scoring Scenario Properties page is displayed when you click the name of a scoring scenario on the Scenarios list page. You can specify the properties when you check out the scenario and click the Edit icon.

Scoring Scenario Properties
Property Description
Name A unique name that can contain only alphabets, numbers, and underscores.
Description The purpose or work of the scoring scenario (optional).
Name (of the score model) A drop-down list of score models that includes all configured score models, in ascending alphabetical order.
Note A score model may be selected more than once in a scoring scenario, on multiple rows. For performance reasons, it will only be executed once per scenario, with the initial score value being reused in subsequent rows.
Score >= The minimum score value (lower cutoff) required to register a decision. This value must be a non-negative integer.
and < The maximum score value (upper cutoff) required to register a decision. This value must be a non-negative integer that is greater than or equal to the minimum value specified in the lower cutoff field.
Decision Decision that will be registered as a candidate decision if the returned score is between the lower and upper cutoffs (inclusive). Decision options include Review, Verify, User 1, User 2, User 3, Re-evaluate, PossibleFraud, Decline, Recommend Decline, Investigate, Recommend Approve, Approve, and No Decision.
