Creating a Decision Tree

Use the Create button in the Decision Trees list page to create a decision tree.
  1. In the Decision Trees list page, click the Create button.
    The Create dialog box appears.
  2. Enter a unique name for the decision tree in the Name field .
    Names must be text strings of 150 characters or less. Names can contain letters, numbers, and underscores.
  3. Optionally, enter a description in the Description field.
  4. Click Create.
    The new decision tree appears in the Decision Trees list page. The decision tree is checked out.
  5. Click the new decision tree to view the Decision Tree Properties page.
  6. Optionally, click the Edit icon to modify properties.
  7. If you want the decision tree to return results, click the Add icon in the Decision Tree Results field.
    Repeat this for all the result values you want to add to the decision tree.
    A default string value appears for each decision tree result.
  8. Customize the Decision Tree Results field values for your business need.
    When you configure an action node in a decision tree to return a result, you can choose one of the decision tree result values from the list.
  9. Create and map variables in the Manage Variables section.
    Condition nodes can evaluate variables and branch the decision tree. For more information see, Creating and Mapping Variables.
  10. To configure condition and action nodes for the decision tree, double-click the default <Unspecified> node to open the Node Properties dialog box.
    Build the decision tree by setting the properties for condition nodes and action nodes. For more information see, Decision Trees: Configuring Condition Nodes and Decision Trees: Configuring Action Nodes.
  11. Click Save and Check In.
    Note When you rename an item or a variable, run the Verify All feature to identify any validation errors. All items that reference the modified item or variable display validation errors. Reconfigure the referencing items to ensure that you can verify them successfully.
