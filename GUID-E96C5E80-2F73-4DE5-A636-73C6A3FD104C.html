Decision Table Editor

Use the decision table editor in the properties page to build and edit the rules that form the decision table.

The Decision Module supports Double-axis layout for the decision table. Each row or column with a header maps to a input variable and evaluates a condition. The cells in the results matrix of the table determine the actions.

The following commands allow you to build the rules in the decision table:
  • Add Row Case
  • Add Column Case
  • Add Condition Row
  • Add Condition Column
The toolbar provides the following commands to manage the rows and columns for the table :
  • Add
  • Copy
  • Delete
  • Cut
Note You cannot delete the row or column in a decision table that contains only one condition row or column. Alternatively, you can delete the decision table from the list page and recreate a new decision table with the required rows and columns.

The editor also contains the ability to print decision table data, and pagination to assist with managing large decision tables.

Use the Rule Profiling icon to attach a sample dataset to a decision table and verify that the rules generate expected results.

Use the Show Filters icon to apply filters to the condition columns to control which rules are displayed. You can also select or enter an expression in a filter to constrain the rules that are displayed.

For more information on how to use the decision table editor, see FICO® Blaze Advisor® business rules management system documentation.

Covered Topics
  • Rule Profiling
    Use the Rule Profiling icon to attach a sample dataset to a decision table and verify that the rules generate expected results.
  • Filtering Rules
    The decision table editor provides a utility to apply filters to rules to customize the display.
  • Inserting a Condition Row and Column
    Add condition rows and columns in a decision table to evaluate inputs. You can evaluate multiple inputs by inserting multiple condition rows or columns.
  • Inserting a Row Case and Column Case
    Insert a row case or a column case to the table to evaluate values for the input variables and define results.
  • Editing a Condition Cell or an Action Cell
    You can edit a cell in the table and modify the values.
