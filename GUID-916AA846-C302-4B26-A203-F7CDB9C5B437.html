Decision Module Rule Processing

With an unlimited number of rules, rulesets, and decision flows possible for each product, the Decision Module needs to set priorities in order to determine the eventual outcome—or decision—for each application. The Decision Module sets priorities using the following mechanisms:
  • Decision levels
  • Severity settings
  • Reason rank order
  • Ruleset results
Covered Topics
  • Decision Levels
    Decision levels determine the final decision to be returned when multiple candidate decisions are registered.
  • Severity Settings
    Rules, rulesets, and products have severity settings that can work together to determine processing results if rule severity usage is activated.
  • Reason Rank Order
    Each rule is associated with a reason, and the Decision Module returns the reason associated with each rule that fires to the calling application.
  • Ruleset Results
    When you configure a rule, you can optionally indicate a candidate ruleset step result to register if the rule fires.
