Creating a Decision Table

Use the Create button in the Decision Tables list page to create a decision table.
  1. In the Decision Tables list page, click the Create button.
    The Create dialog box appears.
  2. Enter a unique name for the decision table in the Name field .
    Names must be text strings of 150 characters or less. Names can contain letters, numbers, and underscores.
  3. Optionally, enter a description in the Description field.
  4. Select the Return type.
    A decision table result type can be of the following data types:
    • Integer
    • Real
    • String
    Note You cannot change the result type of a decision table once you create the decision table.
  5. Click Create.
    The new decision table appears in the Decision Tables list page.
  6. Click the new decision table to view the Decision Table Properties page.
  7. Click the Edit icon to modify properties.
  8. Click Select to map the result to a data field in the object model.
    Browse the object model to select the data field.
  9. Select a data object reference in the context.
    You can use a data object reference to extract the objects for the decision table result.
  10. Create and map variables in the Manage Variables section. For more information see, Creating and Mapping Variables.
  11. Configure the table in the editor using the procedures in the next sections.
  12. Click Save.
    Note When you rename an item or a variable, run the Verify All feature to identify any validation errors. All items that reference the modified item or variable display validation errors. Reconfigure the referencing items to ensure that you can verify them successfully.
