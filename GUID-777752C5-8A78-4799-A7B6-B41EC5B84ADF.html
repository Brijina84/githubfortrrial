Web Service Error Handling

Errors in the Decision Module system can arise in several ways:
  • Functional errors resulting from Decision Module rule service processing. These errors are coded into the structured rule language of the rules project.
  • Exceptions thrown by the Process Server resulting from a failed attempt to call the rule service’s entry point. These exceptions are caught by the Process Server web service.
Two types of exceptions can be thrown by the rule server:
  • Schema validation errors: The rule server throws an exception if the XML document cannot be deserialized into the object model of a rules project. This can be caused by an XML document structure that does not map directly to the object model of the rules project. Schema validation errors are returned as user errors.
  • System errors: Any exception thrown by the rule server other than a schema validation error is treated as a generic “Blaze Advisor system error.”

Interface developers are responsible for managing any new messages generated by modifications to the system and for managing any messages returned outside of the Decision Module. Document any new message numbers.
