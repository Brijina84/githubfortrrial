Rule Properties

The Rule Properties page displays the properties associated with a rule, such as its context, decision reason, severity level, and the conditions under which the rule returns an outcome.

The Rule Properties page is displayed when you click the name of a rule on the Ruleset Properties page of the associated ruleset.

Rule Properties
Property Description
Name A name that is unique within the current ruleset. The name can contain only alphabets, numbers, and underscores.
Description Text that describes the purpose of the rule.
Context The path of the domain model object (selected in the Object Model Browser) on which the rule is defined. The context path is displayed as a sequence of object names, separated by periods, with the first object always being the Application object.
Status A setting that allows you to deactivate or activate a rule without altering its effective date.
Effective date The date range within which a rule is active. Setting this property is optional. If you do not configure this setting, the rule is active immediately.
Ruleset result set to A result value that is registered when the rule fires, and used to determine the next step of the associated decision flow. If multiple results are suggested by multiple rules, the step result with the highest precedence is used.
Use reason A setting that allows you to specify whether you want to use a decision reason. If set to Yes, the rule's Decision reason, Severity level and Reason rank order fields are displayed.
Decision reason The reason that is associated with a decision. If a rule fires, its decision reason is placed on a list along with the decision reasons of other rules that fired. These reasons are included in the decision response.
Severity level A numerical setting that indicates the severity to be added to the ruleset's cumulative severity level if the rule fires. If the cumulative severity level exceeds a threshold value specified for the ruleset, the ruleset registers a decision response. Alternatively, if a severity threshold value is not specified for the ruleset, a decision response is registered when any rule fires.
Reason rank order A numerical setting that represents the order in which this rule's reason appears in the output, relative to other reasons.
Return outcome when Conditions express one or more tests that determine whether a rule will fire.

The Return outcome when settings determine how narrowly or broadly the rule is focused when it evaluates conditions. For example, for the rule to fire, must all conditions be true or just one? Must the conditions evaluate to true for every object that is evaluated, or just one? Should the rule conditions be evaluated for all objects in the rule’s context, or just those that are referenced by a specific data object reference?

Use minimal evaluation A setting that can improve performance by reducing the time taken to execute the rule. Consider a rule that is defined to return an outcome if all conditions are true; if the first condition fails, the rule is not fired. Similarly, when a rule is defined to return an outcome if any condition is true, the rule is fired if the first condition is successful. In both cases, evaluation of the remaining conditions does not impact the outcome of the rule. When this setting is turned on, the conditions that do not affect the outcome are not evaluated.
