Creating Blaze Advisor Functions

Create Blaze Advisor functions to overcome limitations within the data method. For example:
  • To implement efficient functions in the Blaze Advisor Integrated Development Environment (IDE) instead of using the Data Method Editor to write and create Blaze Advisor functions. For information on how to create Blaze Advisor functions, see the FICO® Blaze Advisor® business rules management system Developing Rule Projects Guide.
  • To provide a workaround for data method custom functions that cannot be passed to object model elements.

User-defined data methods can take arguments that are of the string, integer, boolean, or timestamp type. You cannot pass the object types as arguments to the user-defined data methods, but you can use the InvokeBlazeFunction function to do so.

Note You cannot migrate the custom Blaze Advisor functions created in the repository using the Import/Export utility. For a successful migration, an administrator with the proper permissions must manually copy the functions folder you have created.
  1. Connect to the EOS project in Blaze Advisor.
  2. Choose Project > Include Folder or Project > New Folder to create a new folder in the EOS project and enter the name functions for the new folder.
  3. Choose Project > Decision Entities > New Function to create a new Blaze Advisor function in the functions folder.
  4. Open the newly created function and enter the function name. The name must be a unique name within the repository, for example, getRepositoryName.
    The function can only take one parameter of any valid type for the object model, for example, applicationReference as type Application. The return type for the function must always be a string.
  5. Define the function body, which returns a string: 
    if( applicationReference is not null ) then {
 	return applicationReference.DeliveryOptionCode as a string.
}
return "";
  6. Save the getRepositoryName function.
  7. Choose Test > Compile Rule Service to compile the project and fix any errors.
    If there are no errors, a "Compilation succeeded" message appears.
  8. Open the EOS_TEST project.
  9. Include the functions folder in the EOS_TEST project. This step is required for data method testing.
