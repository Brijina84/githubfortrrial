Scenarios

Scenarios are specialized actions that are configured within a decision tree to represent possible outcomes. Scenarios cannot execute directly from decision flow steps.
There are three types of scenarios:
  • Decision
  • Pricing
  • Scoring

Decision scenarios make decisions within an action node of a decision tree. A decision scenario executes from an action node based on the conditional logic defined for that node of the decision tree. Each decision scenario specifies a decision code, decision type, letter ID, queue, and one or more reasons for the decision.

Pricing scenarios make product offers within an action node of a decision tree. A pricing scenario specifies offer parameters (Offer ID, Rate, Amount, and Term) for one or more product offers and results in product offerings. To set additional offer fields, you can define a data method sequence that invokes a data method that sets those fields. You then add the data method sequence to the same decision tree action node as the pricing scenario, immediately after the pricing scenario.

Scoring scenarios make decisions based on scores returned by score models referenced by the scenarios. Each scoring scenario specifies one or more score model information rows. Each row specifies a score model, the low and high ends of its cutoff range, and the decision result to return if the score returned by the score model falls within the specified cutoff range. A scoring scenario may reference multiple score models, and the inclusion of a specific score model triggers the execution of that model. If a scoring scenario references the same score model multiple times, the score model executes only once.

Scenarios can be used across multiple categories and products and, once configured, accessed using the navigation pane of the Item Types tab.
