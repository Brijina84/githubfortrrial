Viewing an Application's History

Use the Application History option to view an application's entire processing history (from the start, through Decision Module evaluation and verification, to the final decision) and the sequence of all user or system activities for an individual application, including any related documentation. Viewing the workflow history of an application can be helpful in determining who has performed system and manual actions on an application and may help you to troubleshoot and resolve problems with that application.
  1. In the Application Information navigation pane of an open application, click Application History.
  2. View the following read-only history information, which appears in date and time order by default (the most recent activity is shown at the bottom):
    • Activity
    • User Name
    • Status Name
    • Creation Date
    • Open Date
    • Close Date
    • Due Date
    • Actions

    The Origination Manager workflow has the ability to generate documents based on templates defined by your system administrator. If documents exist for the application, a Download Letter option appears in the Actions column to open or save the letter that was generated by the Application Processing Module during application processing. The workflow also has the ability to save the request and response between various modules (such as the Decision Module, the Data Acquisition Module, and the Analytic Module) using the application history options. When these modules are called, a View Request/Response link appears in the Actions column so that you can view the XML payload that is transmitted to the other module and received from the other module.

    If an error or exception occurred during processing, an explanation of the error appears at the top of the page and a Retry Activity option appears in the Actions column.

    Note A record's time in Application History indicates the time the record was created and might not reflect the exact time that the status was set.
  3. (Optional) Click the View Application Revision History button at the top of the page to view significant data capture steps, changes to the application status, time of updates, and the user who updated the application. Click the Export Application Revision History button to export the application revision history as a .xlsx file.
  4. (Optional) Click the Export Application History button to export the application history as a .xlsx file.
  5. (Optional) If a request or response file associated with the workflow or application history exists, click the View Request/Response link in the Actions column to review the View Activity Request/Response page. Toggle back and forth between the Request and Response XML file using the Show Response and Show Request options.
    To close the View Activity Request/Response window and return to the Application History page, click Back. To then return to the Application page, click Back to application details.
  6. (Optional) If any application data fields have been marked for tracking, the Check details icon appears in the Actions column. Click the icon to see the details, such as which user made the change, date and time when the changes were made, and the field values before and after the change. At each significant data capture step, the Check details icon appears in the Actions column. Click the Note icon to see the details of the step and the application status, both at the start and end of the step. You can also view the changes to system and manual decisions taken.
  7. (Optional) If documentation associated with the workflow or application history exists, click Download Letter in the Actions column to open or save the letter.
  8. (Optional) If an error or exception occurred during processing, click the Retry Activity option in the Actions column and then click OK in the confirmation dialog box that appears.
