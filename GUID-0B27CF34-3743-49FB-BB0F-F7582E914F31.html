Decision Tables

A decision table allows you to easily define and maintain a large number of rules in a tabular format. You can create and access decision tables from the Item Types tab of the navigation pane or navigate from a Decision Flow that implements the decision table

A decision table can replace many of the rules that compare the same data fields in one or more rulesets making them powerful and efficient tools. The SMW provides pre-configured tables that you can customize to meet your business needs, and you can create new tables.

You can use variables as inputs in a decision table. Use the Manage Variables section in the Decision Table Properties page to map variables to data fields and data methods in the object model. The variables are evaluated when the decision table is run.

You can map a decision table result to a data field in the object model. A decision table result can be of following data types:
  • Integer
  • Real
  • String
After you map the decision table result and variables to evaluate, build the decision table in the editor.
Note The SMW may be slow to display the Properties page or the Visual Differences window when you compare versions of large decision tables such as tables with more than 50 columns and 25 rows. For optimal performance, split the decision table into smaller tables or use the pagination feature in the editor.
Covered Topics
  • Creating a Decision Table
    Use the Create button in the Decision Tables list page to create a decision table.
  • Decision Table Editor
    Use the decision table editor in the properties page to build and edit the rules that form the decision table.
  • Copying a Decision Table
    Use the Save As icon in the Decision Table Properties page to create a decision table from an existing decision table.
  • Deleting a Decision Table
    Use the Delete icon in the Decision Tables list page to delete a decision table that is not required.
