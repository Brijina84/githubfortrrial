Operators

The following characters and words are valid operators (symbols that represent actions) in data method expressions.
Arithmetic Operators
+ Add numbers or combine text
- Subtract
* Multiply
/ Divide
String Operators
+ Combines the two strings on each side of the operator into a single string
““ Default value representing an empty string
Assignment Operator
:= Assign the value on the right to the operand on the left
Object Creation Operator
new Create new object instance of the type specified after the new keyword
Comparison Operators
= Equals
<> Does not equal
< Is less than
<= Is less than or equal to
> Is greater than
>= Is greater than or equal to
Logical Operators
and Evaluate to true if both conditions joined by “and” are true
or Evaluate to true if either of the conditions joined by “or” are true
not Evaluate to true if the condition following “not” is false
Conditional Operators
if...then, endif Execute the statement(s) between “then” and “endif” if condition following “if” is true
if...then...else, endif Execute the statement(s) between “then” and “else” if condition following “if” is true, otherwise execute the statement(s) between “else” and “endif”
End Statement Operator
; End punctuation for a simple statement
Comment Operators
// Begins single-line comment; no end punctuation
/*...*/ Begins and ends multi-line comment
Precedence Operators
( ) Evaluate all operators contained within the parentheses before any others in the statement when used as precedence operator

Evaluate item(s) contained within the parentheses as input(s)

(functions, data methods)

Path and Field Name Operator
. Differentiates object names in a path; also used between an object and a field for field names (Income.Amount)
