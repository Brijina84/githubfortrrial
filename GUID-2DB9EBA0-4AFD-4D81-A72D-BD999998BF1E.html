Adding a Decision Flow Step

Use the Add icon in the Decision Flow Properties page to add steps to a decision flow. The Add icon appears in the editor.

You must check out the decision flow and edit the decision flow to add steps.

  1. In the Decision Flow Properties page, to add a step, drag and drop the plus icon on to the canvas.
    The New Step dialog box appears.
  2. Enter a unique name for the step.
  3. Select an Implementation Type from the drop-down list.
  4. Select an Implementation Name from the drop-down list.
    The drop-down list displays the available items of the implementation type.
  5. Select Make first step if you want the step to be the first step in the decision flow.

    The first step in the decision flow is blue.

  6. Click Apply to ensure field values are acceptable.
    If the field values are not acceptable, the system displays an error message.
  7. Click Save.
    The Step Details dialog box disappears.
  8. Repeat this procedure to add all the steps required in the decision flow.

After you add the steps required in a decision flow, you can connect the decision flow steps. Use the Connection icon, or the Transition Information section in the Step Details dialog box. Decision flow steps can transition from one step to another step or to multiple steps. Multiple steps can transition to one. Loops are also valid. If no transition is specified, the default transition is Stop Processing.

Note Controlling loop process execution must be achieved in an implementation target such as a rule or data method. The step itself does not control loop execution. If you do not provide a control, it is possible to set up an “infinite loop” where the process keeps running endlessly.
