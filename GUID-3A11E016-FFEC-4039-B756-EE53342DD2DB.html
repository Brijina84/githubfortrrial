Product Selection Tree Profiling

Use product selection tree profiling to see how data flows through the product selection tree logic and how the actions are distributed for a given sample dataset.

You can use profiling for your product selection trees. When the profiler runs, the condition nodes in your product selection tree evaluate each record in a dataset. The data that matches the conditions and actions is calculated and displayed in the product selection tree editor as a record count.



Product Selection Tree Displaying in Tree View

Covered Topics
  • Dataset Requirements for product selection tree Profiling
    The dataset used for product selection tree profiling must be a CSV (comma-separated values) file that conforms to specific requirements. You can create your own dataset in an external application or generate a sample dataset and modify the values.
  • Generating a Sample Dataset for Product Selection Tree Profiling
    Generate a sample dataset based on the current product selection tree. The generated .csv shows the correct format for the dataset and includes a column header and column for each condition in a product selection tree. After the sample dataset generates, you can modify the values as required.
  • Previewing and Running Data Profiling
    To run the product selection tree profiler, attach a .csv dataset to run against the conditions in the product selection tree. Only one dataset can be attached to a product selection tree.
