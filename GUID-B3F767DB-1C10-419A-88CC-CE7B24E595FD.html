Rules

A rule is the combination of one or many conditions and a resulting outcome that implements organizational decisions or policies. Each rule may have one or more conditions to evaluate. Each rule statement is in the form: For "some attribute" in the domain model or some data method, if "some condition" is true, then do "some action". For example, if you want a rule to check if the applicant has lived at the current address less than 24 months, it might contain a condition like this:
Context (Data Object) Field or Data Method Operator Value
Application.creditApplication.personalApplicant monthsAtCurrentAddress is less than 24

In the domain model delivered with the Decision Module, the personalApplicant object includes the monthsAtCurrentAddress field. Whenever this rule is triggered, the associated "reason" is returned to the system if the rule fires (evaluates as true).

If you want to include a field that exists in your institutionâ€™s credit application but is not in the domain model, you need to set it up as a user-defined field or add the field to the domain model before you can use it in a rule. If you want to use data derived from a domain model field in another object, you need to define a data method that generates the derived result or retrieves the data, and then use that data method in the rule condition.
