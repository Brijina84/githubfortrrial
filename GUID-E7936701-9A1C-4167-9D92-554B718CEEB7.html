Creating Data Methods: Overview

Before building a data method, you typically gather relevant information and create an outline of the process.

The following is a summary of the steps involved:

  1. Decide what you want your data method to do. This will become the data method description.
  2. Decide what type of data method you need to create. Will it return a value (dm_) or an object (dmo_), or will it perform work but not return anything (dmw_)?
  3. Decide on a name for your data method, following the naming conventions in Data Method Naming Conventions.
  4. Using the Object Model Browser, view the items (objects, fields, data object references, and data methods) that are available within the desired context. For example, if you are going to write a data method on the Application.creditApplication.personalApplicant.applicantFinancial object, look at the items in the Object Model Browser under the applicantFinancial object. If you need to access an item that is not on the path of the current context, you will need to use a data object reference or another data method to do so.
    Note You can also use the Object Model Browser search feature to help identify an appropriate context for a data method when it is not obvious. For example, if you are not sure where the Birthdate field is, you can search for it and then set the context to the object that contains the field.
  5. Using the guidelines in Using the Data Method Expression Language, determine a valid structure and syntax for your expression statement type. What kinds of actions or operations need to take place? What kinds of statements (assignment, conditional, comparison) do you need to create?
  6. Determine the expression items (inputs, locals, enumerations, fields, user-defined fields, functions, data object references, and so on) you will need to use or define.

When you have completed these steps and defined any prerequisite items, you are ready to begin creating your data method.
