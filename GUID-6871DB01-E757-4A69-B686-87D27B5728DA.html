Champion-Challenger Decision Flows in Decisioning

A product strategy can have one or multiple decision flows that is run based on the conditions. You can introduce decision flows to a product strategy to challenge the existing champion decision flow.

In order to optimize credit policy efficiency and effectivness, you can design and evaluate product strategies with multiple decision flows for each of your products. For example, you might consider introducing new decision flows for your products if interest rates have increased, or if a merger has altered business demographics. You might want to introduce a new decision flow to a product strategy to test the outcome on a small percentage of transactions before deciding whether to replace your existing main decision flow.

Consider the following two types of decision flows in a product strategy:
  • Champion decision flow
  • Challenger decision flow

A champion decision flow is a proven decision flow. If the product has more than one decision flow, the champion is the most established one and typically handles the decisioning for a majority of the applications for that product. Compare the champion decision flow to the challenger decision flow.

A challenger decision flow can have a potential improvement in comparison to the current champion decision flow. The challenger decision flow handles making decisions for a percentage of applications for the product until it is either judged less effective and discarded, or deemed to be better than the current champion. In the latter case, the challenger replaces the current champion decision flow. A product may have multiple challenger decision flows, and each challenger may have an effective start or end date.

Note A challenger should always remain in place long enough for it to have an observable impact. Your initial champion-challenger points of comparison need to be preserved as much as possible. Changes made too soon after implementation can prevent fair evaluation.

Several preconfigured mappings allow you to implement champion-challenger decision flows with effective start and end dates without having to create the mappings yourself. These appear in the Node Properties dialog box of the Product Strategy Properties page.

Note When a strategy selection random number is provided in the input XML, this value is used in the product strategy if the strategy uses the random number mapping. Otherwise, the Decision Module automatically generates a random number for the mapping.

The Decision Module also provides a number of decision and scoring reports that can help you analyze and improve your strategies.

Covered Topics
  • Example: Champion-Challenger Decision Flows
    You can use preconfigured mappings to implement champion-challenger decision flows.
  • Creating a Product Strategy
    Use the Create button in the product strategies list page to create a new product strategy.
  • Assigning a Product Strategy to a Product
    Use the Product Properties page to assign a product strategy to a product.
  • Decision Flow Selection Tree Tasks
    You can perform tasks, such as insert, copy, edit, clear, assign work on a selected tree node.
  • Creating and Mapping Variables
    Create and map variables to data fields in the object model. Variables determine the fields that condition nodes evaluate when the tree is run.
  • Inserting Splits
    Add a split to a decision flow selection tree when you want to create a set of branches for condition nodes.
  • Inserting Levels
    All condition nodes on a single level represent a segment of the same decision variable. When you create a decision flow selection tree, it does not contain any levels. Add a level using the Insert Level command in the Edit drop-down menu. You can also insert a level at the same time you insert a split using the Insert Split command.
  • Copying a Product Strategy
    Use the Save As icon in the Product Strategy Properties page to create a product strategy from an existing product strategy.
  • Deleting a Product Strategy
    Use the Delete icon in the Product Strategy list page to delete a product strategy that is not required.
