Analytic Data Mart

Your decision service execution automatically logs event data to the Analytic Data Mart (ADM). The ADM captures and stores event data each time a client application invokes a decision service running on the Decision Management Platform (DMP).

ADM captures data and makes it available for reporting and analysis. At its core, ADM consists of two data stores-- the event store and the report store. The event store is a log of past decisions and the report store is an analytical data store. Together, they enable you to improve decision models and track progress against key strategic and operational goals. Each time a decision service is invoked, event data about the invocation and environment is captured and persisted in the Analytic Data Mart. Depending on the settings you choose, the input and output payloads of the decision service requests may also be captured. The data can be accessed from a dedicated Tableau Server site, where you can develop the reports.

The data logged in the Analytic Data Mart is available in Decision Testing and can be used to execute against new versions of the decision service. You can also import data from batch processing or web service deployments into Decision Testing. When you import the ADM data into Decision Testing, you can export it from Decision Testing as a CSV file.

You can build reports on your service execution. You can use Decision Testing Analyzer to pull in event data into Decision Modeler for analysis. However, if you want to generate Tableau reports for a decision service based on the input and output data collected in the ADM tables, you must purchase a Tableau license.

Make selections for Data Capture Settings to control what event data is persisted to the Analytic Data Mart. You can make selections for each lifecycle environment (Development, Staging, or Production). The settings control:
  • Whether or not payload data from decision service invocations are captured.
  • Whether or not internal events are captured for all decisions in a DRD, including decisions that are not part of the output.
Note If you have access to the integrated Tableau Server, you can see default reports, and create your own worksheets, from the data sources. If you do not have access to Tableau Server, contact your FICO representative.

Run Reports

When you invoke a decision service, the data is stored by the Analytic Data Mart but is not immediately available in Tableau. By default, event and report data is sent to Tableau once every 24 hours. The RUN NOW button allows you to run the reports manually and access your data in Tableau on-demand.

RUN NOW button

If the Enable checkbox is selected, the reports will run once every 24 hours and the RUN NOW button will be available. If you require increased performance, contact FICO support.
