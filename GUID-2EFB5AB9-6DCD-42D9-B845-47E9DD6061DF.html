Verify Items

Verify the items configured for the business strategy to ensure that they are valid. Perform verification frequently to ensure your strategies are ready to execute.

When you verify items you can uncover certain conditions that could prevent strategies from executing, such as when a rule references a data method that does not exist.

Use the Verify icon in the properties page to verify an item. Use Verify All from the Tools tab to verify all the items configured for the business strategy, and verify all items of an item type .

When an item is verified, the Decision Module verifies all the dependent items. It does not verify items that use the item. For example, if you verify data method A, the Decision Module verifies all data methods, data object references, and other items that data method A and all its dependent items use. It does not verify rules, data methods, or other items that use data method A.

Verification process also detects unused items and circular references, such as item A refers to item B, which refers to item A.

Covered Topics
  • Verification Report
    When you verify items, the Decision Module generates a report with the verification status of the item and error details.
  • Verifying an Item
    Use the Verify icon in the properties page of any item to verify the item.
  • Verifying All Items
    Use the Verify All icon in the Tools tab to verify all items in the repository or all items of an item type.
  • Item Verification Reference
    The Item Verification reference table lists the dependent items that the Decision Module can verify when you verify an item.
