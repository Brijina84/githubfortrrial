Using the Data Method Expression Language

The Data Method Expression Language is a proprietary language designed for creating data method expressions. It allows the business analyst to implement complex data methods for use in strategies.

To write your own expression statements using the expression language, you need to familiarize yourself with its syntax and conventions, and with the operators, keywords, and functions that are available to you.

A data method expression statement is a combination of characters and symbols that represents a value and conforms to the Data Method Expression Language syntax and conventions. Data methods must have at least one expression statement—the initial statement. They may also contain:
  • Additional simple statements
  • Conditional statements: IF this statement is true, THEN do this action
  • Processing loops: FOR EACH object, DO this action

All expression statements consist of at least one operand and can have one or more operators. Operands are the objects or values that are manipulated, and operators are the symbols that represent specific actions. For example, in the phrase “y + 7”, y and 7 are operands and + is the operator. Simple statements must be followed by a semicolon. Conditional statements and processing loops must be followed by a space, tab, or carriage return, and it is highly recommended that they be followed by a new line for clarity.

  • Simple Statements
    Simple expression statements—statements that do not contain conditional statements or FOR loops—can be any of the following:
  • Assignment Statements
    A simple assignment statement contains at least one expression containing the := assignment operator. It may contain additional conditional expressions.
  • Function Calls
    Data method expressions can contain function calls—operations that invoke a function. A function is a named procedure in a data method that performs a specific task.
  • Data Method Calls
    Expressions can contain data method calls—operations that invoke another data method.
  • Conditional Statements
    Conditional statements are statements that perform different actions depending on the particular inputs they are evaluating.
  • Processing Loop Statements
    Processing loops are instructions that repeat until some condition is met. They allow you to incorporate fields and data methods from non-context domain model objects into your current data method, enabling more powerful data methods to be written.
  • Data Methods that Return Values
    Within the data method’s expression, you can specify what value will be returned by the method. In addition, you can make assignments to the local variables.

    Dot notation is a convention used in data methods to indicate an object’s path. Each child object is separated from its parent by a period.
  • Keywords
    A keyword is a word that is reserved for specific uses because it has a special meaning. Keywords can be commands or parameters.
  • Precedence in Complex Expressions
    Precedence is a characteristic of operators that indicates their evaluation order when multiple operators appear in a complex expression. Operators with high precedence are evaluated before operators with low precedence.

    Comments may be embedded in the logic of expressions to help the reader understand the processing being done.
