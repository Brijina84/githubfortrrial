Data Method Naming Conventions

It is recommended that you use consistent naming conventions to facilitate data method identification. The following are suggested naming conventions:

Prefix—An abbreviation for the type of data method, followed by an underscore:

dm_ Value data method

dmo_ Object data method

dmw_ Work data method

Context—A word or abbreviation with initial capitalization, for the context of the data method, such as dm_Root (for the root object, Application) or dm_App (for the Applicant object).

Identifying Text—Additional words or abbreviated text with initial capitalization that suggest the purpose of the data method, such as dm_Liability_isBankruptcy. In this name, “is” represents a true/false (boolean) operation. This data method returns an indicator that tells whether there is a bankruptcy.

Note If you use initial capitalization for name parts, you avoid conflicts with lowercase names, such as keywords.

The following table shows a breakdown of the name of a data method example called dm_RootDebtToIncomeRatio.

Naming Convention Example
Name Component Description
dm_ This is a value data method.
Root The domain model context of this data method is the Application object.
DebtToIncomeRatio The purpose of the data method is to calculate the ratio of monthly debt to monthly income for items listed in the application.

You may create additional conventions to further identify your data methods. The key to workable naming conventions is consistency.
