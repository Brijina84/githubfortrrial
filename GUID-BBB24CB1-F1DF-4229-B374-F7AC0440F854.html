Precedence in Complex Expressions

Precedence is a characteristic of operators that indicates their evaluation order when multiple operators appear in a complex expression. Operators with high precedence are evaluated before operators with low precedence.

The following table lists operator precedence in data method expressions.

Numeric Operator Precedence
Precedence Level Operators
1 ( )
2 *

/

3 +

-

For example, the multiplication operator * has higher precedence than the addition operator +, so the expression 2+3*4 equals 14, not 20.

You can override precedence rules by surrounding parts of an expression with parentheses. For example, (2+3)*4 would evaluate to 20.

There is an additional aspect of precedence—the use of multiple sets of parentheses—that comes into play when there are multiple operations. Keep in mind the following basic rules when creating complex expressions:
  1. Operations are evaluated from left to right.
  2. Inner parentheses are evaluated before outer parentheses (the more parentheses that surround an operation, the higher its precedence).
  3. At the same precedence level, multiplication and division are performed before addition and subtraction.
So, in the expression ((50 + 3) * ((100 - 27) * (80 - 45))), operations would be performed and evaluated in the following order:
  1. 50 + 3 = 53
  2. 100 - 27 = 73
  3. 80 - 45 = 35
  4. 73 * 35 = 2555
  5. 53 * 2555 = 135,415
Note When parentheses are used to indicate precedence, they must always occur in pairs. If you have a different number of left and right parentheses, a validation error is reported.
