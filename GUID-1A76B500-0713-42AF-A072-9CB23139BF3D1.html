Best Practices for Managing Object Model Changes

When you use the SMW to manage Object Model changes, consider the following best practices based on your user role:

System Administrator Role

  • Ensure that the SMW users are informed about the changes to the Object Model when you promote a new version of the schema.
  • Use either the SMW or the Blaze Advisor IDE to modify the Object Model. You cannot modify the Object Model schema within the SMW if the Object Model is checked out using the Blaze Advisor IDE.
  • Avoid removing or renaming elements during the strategy component design phase. Removing or renaming schema components that may be used in the configuration can lead to verification errors.

Guest Role

  • Navigate to Manage Object Model in the Tools tab. If there are outstanding Object Model updates, click the Update link to update the Object Model.
  • After you update the Object Model run Verify All on the entire repository to ensure there are no verification errors.
  • Verify that you are using the latest version of the Object Model schema by viewing the version history of the schema files.

For more information on user roles and permission types see, FICO Origination Manager Decision Module Administrator's Guide.
