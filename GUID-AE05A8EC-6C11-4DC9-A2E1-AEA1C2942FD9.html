Data Method Properties

All data methods, whether simple or complex, have the same basic properties and structure.

Name—A unique name. Data method names begin with a prefix representing their data method type, as described in Data Method Naming Conventions.

Description—A brief description of the purpose of the data method.

Context—The full path of the object on which the data method will be executed. The context path is displayed as a sequence of object names, separated by periods, with the first object always being the Application object.

Result type—The type of result returned by the data method. The data method result type can be a data type (boolean, numeric, string, timestamp, date, time, or duration), or a domain model object. A dat maethod expression may or may not have a result.

Save history—A setting that directs the system to save the data method history. This can be very helpful when testing data methods.
