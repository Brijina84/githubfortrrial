Data Methods

A data method is a procedure associated with a specific type of object within the domain model. It performs one or more operations when executed, and may produce new information or take an action such as setting a field.
Data methods can be:
  • Used to compute, manipulate, set, select, or return values using standard mathematical operations
  • Grouped into data method sequences
  • Used in rules or score models
  • Used as inputs for decision tables and decision trees
  • Nested within other data methods

Data methods are created on the Data Methods list page. When you edit a data method, you can include inputs and locals, and create a data method expression describing the work to be done.

A data method expression is a statement or group of statements written using a specified syntax. The expression defines the operations to be performed when the data method executes. Expressions can contain inputs, local variables, operators, functions, domain model objects and their fields, data object references, other data methods, and specific syntactic characters.

A data method has a context, meaning that it executes at the level in which the object that it references resides. For example, if you want "total income" to be summed into one value for a group of applicants (primary and co-applicant, for example) for an application, then you could create a "total income" data method at the application level. Or, if you want "total income" to be summed for each applicant individually for an application, then you could create the data method at the applicant level, producing a total income for each applicant.
