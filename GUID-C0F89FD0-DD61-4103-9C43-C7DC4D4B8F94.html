Using User-defined Fields

The UserDefinedField object enables you to create user-defined fields to hold data dynamically created by the Decision Module for use in processing the current request.

In effect, user-defined fields act like global variables in the context of a particular request. Once set, they can be used by other rules and procedures that run on the request.

All but the following types of objects in the Decision Module domain model contain UserDefinedField children:

  • DataMethodHistory objects and their descendants
  • UserDefinedField objects and their descendants

An object can have multiple instances of UserDefinedField objects; instances can be created or destroyed programmatically.

No schema changes are needed to add or modify values of UserDefinedField objects.

UserDefinedField elements have this structure:
  • <ParentElement>/UserDefinedField
  • <ParentElement>/UserDefinedField/Name
  • <ParentElement>/UserDefinedField/DataType
  • <ParentElement>/UserDefinedField/Value
<ParentElement> is a placeholder for any element that can contain a UserDefinedField object. The value of the Value element must be consistent with the data type listed in the DataType element. The Value element can have the following data types:
boolean real
date string
duration time
integer timestamp
Useful data method functions for converting to other data types include:
  • Val() to convert a string to a numeric type.
  • TimestampFromStr() to convert a string to a timestamp.
  • TimestampToDate() to convert a timestamp to a date.
