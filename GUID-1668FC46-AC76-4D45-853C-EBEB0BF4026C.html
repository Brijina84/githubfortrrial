Resolving Fraud for Applications for Synchronous Consumer Line of Business

To resolve the fraud application issue, your system has to be configured to perform velocity check when a new consumer credit application is submitted and the velocity check event must be included in the synchronous workflow used for application processing. When the system is configured for velocity check, the system checks to make sure that the new application is not a fraud application. Fraud resolution is a human task to determine whether an application should be flagged as a fraud application and if the application should be withdrawn. The workflow stops till the human activity of fraud resolution is complete.
Because velocity check is part of the Synchronous Consumer LOB, if any of the velocity check parameters match, the application moves to the Pending Fraud Resolution status and the Fraud Resolution page opens.To resolve the application's status, review the following information:
  1. To resolve the application status, review the following information:
    • The information at the top of the Resolve Application <Application ID Number> for <Applicant Name> page.
    • The information in the Matched Applications pane, which provides a summary of the information for the application or applications.
  2. Evaluate all applications in the list and perform one of the actions given below:
    • If the application is a fraud, click Fraud. The system sets the status of the application to Pending Withdrawal Archive.
    • If the application is not a fraud, click Not Fraud. The application processing continues as per the workflow.
