Severity Settings

Rules, rulesets, and products have severity settings that can work together to determine processing results if rule severity usage is activated.

When a rule executes and its conditions are met, its numeric severity level is added to a cumulative severity level for the current ruleset, and its reason is added to a reason list for the ruleset.

When ruleset severity threshold is enabled, after the ruleset completes processing all rules, if the cumulative severity level for the ruleset exceeds the ruleset severity threshold, the ruleset registers a candidate decision. This decision includes the decision level (decline, approve, and so on) for the ruleset, and a list of reasons for all rules that fired. If the ruleset’s severity threshold is not exceeded, the ruleset does not register a candidate decision, but the reasons produced by the rules in that ruleset are still registered and will be returned to the calling application with a decision result of Rule History.

When product severity is enabled for a product, the Decision Module initializes the product’s cumulative rule severity level to zero before the product’s decision flow executes the first step. Each time a ruleset completes execution, the cumulative severity of all rules that fired in the ruleset is added to the product’s cumulative rule severity level. If the product’s cumulative rule severity level exceeds the product’s rule severity threshold, the decision flow is immediately terminated. Otherwise, processing continues with the next decision flow step. If product severity is disabled for a product, it does not maintain the product’s cumulative rule severity level.

Decision trees, like rulesets, may also register candidate decisions. When a decision tree executes a Select Decision action or executes a decision scenario or scoring scenario that returns a decision, it registers a candidate decision with the specified decision level and reason. Candidate decisions for all decision trees are returned to the calling application, along with their associated reasons.

When processing terminates, either because the decision flow processing terminates normally or because the product’s severity level threshold has been exceeded, the Decision Module makes a final determination of the decision result:
  • If only one candidate decision has been registered, the final decision result is the decision level of that candidate decision.
  • If multiple candidate decisions have been registered, the final decision result is the decision level that is the most conservative.

The final and all candidate decisions are returned to the calling application, along with a reason list for each decision.
