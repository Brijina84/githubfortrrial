Rule Context

A rule’s context is the full path of the domain model item on which it is defined.The context path is displayed as a sequence of object names, separated by periods, with the first object always being the Application object.

When creating rule conditions, you are not limited to using only the fields and data methods that are defined in the same context as the rule. The Decision Module allows you to utilize fields and data methods defined for any object along the context path; so you can include information from multiple objects in your rule conditions.

There are two basic requirements for rule conditions:
  • All data methods and data fields that you use in a rule’s conditions must be on one path.
  • The rule’s context must be an object whose path contains this path.

For example, if the context for a rule is the Application.creditApplication.personalApplicant object, you can use any field or data method that is associated with the Application, creditApplication, or personalApplicant objects when creating the rule’s conditions.

When data that needs to be evaluated exists on another path, you can create a data method to access it. Data methods are the only Strategy Management Workstation components that can perform this “joining” of objects from different paths. For example, if a rule's context is Application.creditApplication.personalApplicant, and you want to access a field under Application.applicant, you must create a data method for that object to bridge the context gap, by bringing the required data to the Application, creditApplication or personalApplicant context.
