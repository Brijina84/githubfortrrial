Data Method Calls

Expressions can contain data method calls—operations that invoke another data method.
A data method call has the following syntax: 
Data_Method_Name(method_inputs)

where method_inputs is an optional comma-separated list of inputs.

A data method call may optionally include an ordered and defined list of inputs. If the data method call has no inputs, the parentheses are not used: 
Data_Method_Name

The following is an example of a data method that contains a data method call.

Data Method Call
Item Value
Name dm_getCurrentDebtToIncomeRatioForDecisionParty
Description Returns the current debt to income ratio with housing for the decision party.
Context Application.creditApplication.personalApplicant.applicantFinancial
Result Type numeric
Data Method dm_getAppCurrentDebtToIncomeRatioWithHousing, and dm_getCoAppCurrentDebtToIncomeRatioWithHousing
Expression result := 0; 
if(creditApplication.decisionParty = "DECISION_PARTY_TYPE_PRIMARY") then
result := dm_getAppCurrentDebtToIncomeRatioWithHousing;
endif
if(creditApplication.decisionParty = "DECISION_PARTY_TYPE_SECONDARY") then
result := dm_getCoAppCurrentDebtToIncomeRatioWithHousing;
endif

The dm_getCurrentDebtToIncomeRatioForDecisionParty data method calls one of two data methods, depending on the decision party type: dm_getAppCurrentDebtToIncomeRatioWithHousing or dm_getCoAppCurrentDebtToIncomeRatioWithHousing, and the data method called returns a debt to income ratio accordingly.

The data method expression reads as follows:
  1. Assign result an initial value of 0 (because the result type is numeric).
  2. Execute the dm_getAppCurrentDebtToIncomeRatioWithHousing or dm_getCoAppCurrentDebtToIncomeRatioWithHousing data method and assign the resulting value to result.
