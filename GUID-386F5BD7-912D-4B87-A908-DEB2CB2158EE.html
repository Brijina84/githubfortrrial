Adding a Rule to a Ruleset

Use the Add icon in the Rules section of the Ruleset Properties page to add rules to a ruleset.
  1. In the Rulesets list page, select the ruleset to which you want to add a rule.
    The Ruleset Properties page appears.
  2. Make sure you can edit the ruleset:
    • If the ruleset has been checked in, click Check Out.
    • If the ruleset has not been checked in yet, click Edit.
  3. In the Rules section, click the Add icon.
    A rule is added with default values for Name (rule1, rule2, and so on) and Status (Active).
  4. To update the default values and define other properties for the rule, click its name.
  5. In the Rule Properties page, enter a name that is unique to the ruleset.
    Names must be text strings of 150 characters or less. Names can contain letters, numbers, and underscores.
    Note Certain lowercase words are Blaze Advisor reserved words and cannot be used in names. For a list of these words, see Avoiding Name Conflicts. You can avoid this problem by using mixed-case names.
  6. (Optional) In the Description field, enter text that describes the purpose of the rule.
  7. The default context for the rule is Application. If you want the rule to be executed on a different object, click Change.
    1. In the Object Model Browser, select an object.
      You can browse to the object or search for it.
    2. Click Select.
  8. By default, the rule is active. You can make it inactive by changing the Status option.
    This option allows you to activate or deactivate the rule without setting or modifying its effective dates.
  9. (Optional) To set a start or end date for the rule, click the from or to arrow for Effective date, then click Select Date, and select a date.
  10. Select a value from the Ruleset result set to list.
    When the rule fires, the result value is registered and used to determine the next step of the associated decision flow. Results appear in this list only if they have been configured first for the ruleset.
  11. In the Use reason field, specify whether you want to use a decision reason.
    If you select Yes, the Decision reason, Severity level, and Reason rank order fields are displayed. The following three steps are applicable only if you select Yes.
  12. To return a reason in the decision response when the rule fires, enter a reason in the Decision reason field.
  13. Enter an integer in the Severity level field to indicate the severity level to be returned by the rule if it is executed. This value is added to the ruleset's cumulative severity level. This field is ignored it a severity threshold is not specified for the ruleset.
  14. Enter an integer in the Reason rank order field to indicate the rank order of the rule’s reason, relative to the reasons of other rules (within the ruleset) that execute.
  15. To define the rule’s scope, follow the steps in Setting the Rule’s Scope.
  16. To define the rule's conditions, follow the steps in Adding and Configuring Rule Conditions.
  17. Click Save.
