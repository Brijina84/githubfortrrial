Business Strategies Workbench User Interface

The Business Strategies workbench has a intuitive user-friendly interface that allows you to create and maintain the components, such as rulesets, decision tables, decision flows, and so on.
Covered Topics
  • Left Navigation Pane
    The Business Strategies drop-down menu shows Item Types, Test, and Tools. Each menu item has a left navigation pane whose options provide access to various aspects of the Business Strategies functionality. The functionality that you can access and use depends on your role and privileges.
  • Private Workspaces
    The Strategy Management Workstation has a configuration repository that stores the items used to implement business strategies. Each SMW user has a private workspace that is connected to the configuration repository. Users can create, edit, and delete items within their private workspaces.
  • List Pages
    A list page is displayed when you click an item type such as Decision Flows or Data Methods in the navigation pane. All list pages contain a Create button, a text box for search, a panel of action icons, and a table listing the items available for viewing and selection.
  • Properties Pages
    Configure item type properties in the properties page. A properties page can contain editors and tool bars to modify properties.
  • Item Search
  • Manage Variables
    Use the Manage Variables section in a properties page to map variables to data fields available in the object model. Several item types such as score models, product selection trees, decision trees, and decision tables use variables.
  • Where Used Search
    Use Where Used search to find where items are used in your business strategies.
  • Verify Items
    Verify the items configured for the business strategy to ensure that they are valid. Perform verification frequently to ensure your strategies are ready to execute.
  • Deleting Items
    Use the Delete icon on the list page to delete items that are no longer needed.
  • Renaming Items
    Rename an item in its properties page.
  • Versioning Workspace Items
    The SMW controls user access and implements versioning by enabling users to check in and check out items. Versioning allows multiple users to safely work on items without overwriting another personâ€™s work.
  • Accessibility
    The Decision Module provides keyboard support and other accessibility features to navigate the pages and fields of the SMW.
  • Printing
    Use the print icon that appears in the editors for item types such as decision trees, decision tables, product strategies, product selection trees, and score models to print the details in the editor.
  • Manage Object Model
    Use the Manage Object Model option in the Tools tab to view the Object Model schema and its version history.
  • Logging Off
    There is only one correct way to exit the SMW. It is important that you log off at the end of each session before closing the window or proceeding to a different URL so that your connection to the server is released correctly.
