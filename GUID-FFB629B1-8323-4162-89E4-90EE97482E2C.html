Handling Data Change In Application BOM Fields

Data change in certain entities or fields of the application must be detected because a data change in these fields might require the application processing workflow to start again or the application to be rescored. Use the Data Change Actions page to add the list of fields that would require either a restart of the application processing workflow if a field is edited. Then, select the list of fields that require rescoring the application if a field is edited.
  1. In the administration navigation pane, click Data Change Actions.
  2. In the Require Re-execution area, perform the following actions:
    1. Add fields by selecting them individually from the Available list, and then clicking Add. To add all fields, click the Add all button. To add more than one field at a time, hold the Ctrl key down, select the fields, and then click Add.
    2. Use the Remove option to remove fields from the Selected list. Click Remove all to remove all fields from the Selected list.
  3. In the Require Rescore area, perform the following actions:
    1. Add fields by selecting them individually from the Available list, and then clicking Add. To add all fields, click the Add all button. To add more than one field at a time, hold the Ctrl key down, select the fields, and then click Add.
    2. Use the Remove option to remove fields from the Selected list. Click Remove all to remove all fields from the Selected list.
  4. Click Save.
