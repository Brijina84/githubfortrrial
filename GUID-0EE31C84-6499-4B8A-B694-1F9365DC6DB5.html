Validating Score Models

The Decision Module provides two ways to verify that the values entered in the score model are valid: characteristic-level validation and model-level validation.
With characteristic-level validation, you receive immediate feedback if an invalid value is entered in a characteristic. If the score model detects an invalid value, the following error indicators are shown:
  • An error message appears immediately for data type errors.
  • The font color in the Range column for a given characteristic changes to red.
  • The Show Validation Errors option is available when you right-click in the Characteristics section.
The system detects the following types of errors:
  • Data type errors, such as entering a string value in a range cell that only allows a real value.
  • Overlapping value errors, such as entering a value in the first bin that overlaps a value in the second bin.
  • Duplicate expression errors, such as using the same expression format and range in two bins.
  • Boundary errors, such as entering a value in an upper bound that is less than the value in the low bound.
  • With model-level validation, you use the Validate Model icon from the toolbar in the Characteristics section to receive a validation message for the entire score model.
