Error Handling

A response from the Decision Module lists the type of outcome by setting a status attribute in the Outcome node of the envelope. Three statuses are possible: SUCCESS, ERROR, or WARNING. The outcome status can appear at the envelope, service, and operation levels.

Error information is given in the envelope Outcome/Message/MessageDetail node. The level at which the information appears depends on how far the transaction was able to proceed.

Decision Module warning and error messages fall into three categories:

  • Module-level, related to problems with tenant or system user information
  • Decision Module web service messages, related to the rule processing for the decision request
  • Process Server messages, related to configuration of the Process Server

Module-level errors have an error code prefixed with FIOM- in an Outcome/Message/MessageDetail node in the envelope. Errors from the Decision Module web service or the Process Server are listed both in a MessageDetail node and in the Application/MessageList node in the output payload.

Message numbers 10000 through 10999 are reserved for functional errors, which are placed directly into the Application/MessageList node by the rule service.

Message numbers 11000 through 11999 are reserved for other types of errors.
