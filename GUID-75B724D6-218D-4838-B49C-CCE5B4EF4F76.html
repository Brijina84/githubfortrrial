Testing in the Business Strategies Workbench

The Business Strategies Workbench includes a feature that allows you to test components and business strategies. Test components and business strategies with different scenarios to ensure that the decisions returned suit your business practices.
Use the Test menu item in the workbench to test business strategies. The Test menu allows you to perform the following actions:
  • Create a base application in XML format.
  • Define test scenarios with input test data and output test data.
  • Execute test scenarios and review test results.
  • Save test scenarios and retest in future.
The Test tab includes the following options to test components and business strategies:
  • Test Execution. Add input use cases and expected output for components and strategies. Run tests and view test results against expected output.
  • Base Applications. Create, import, and manage input applications.
  • Test Scenarios. Create and manage input test data and output test data.

Validate test results against expected output to ensure that the decisions returned suit your business practices. When test results do not match the expected output, review and reconfigure components and business strategies.

Covered Topics
  • Base Applications
    A base application is a standard input application in XML format.
  • Test Execution
    Test components and strategies in the SMW with different test scenarios. You can retrieve a base application and add use cases to test.
  • Test Scenarios
    Test scenarios define the different use cases to test components and strategies. You can save the test scenarios in the private workspace in the SMW application server and retrieve them to retest components and strategies.
  • XPath Expressions for Input Test Attributes
    You can enter input test attributes as XPath expressions to create test scenarios.
  • Managing Base Applications and Test Scenarios
    Use the Test tab in the SMW to manage base applications, manage test scenarios, and test components and strategies.
  • Example: Application Processing for Revolving Category
    Use a base application and test scenarios to evaluate policies designed for processing applications for a product category.
  • Test Icons
    Use the icons in the Test tab to manage base applications and test scenarios.
