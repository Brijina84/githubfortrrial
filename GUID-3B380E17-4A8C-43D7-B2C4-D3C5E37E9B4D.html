Testing a Data Method

After you have created, validated, and saved a data method, use the Test icon on the Data Method Properties page to test the data method.
  1. On the Data Method Properties page, make sure that the conditional logic in the Expression Editor is what you want to test. If required, edit the data method expression, and click Save.

    If you do not save your changes at this point and proceed to test the data method, you will be prompted to either save your changes, or test the data method against the last saved version.

  2. Click the Test icon in the toolbar.
    The Test Data Method window appears.
  3. If the data method accepts input parameters, specify them.
  4. The Create Test Data area is populated with XML code based on the data method's definition. To test the data method, you can use this data or use a base application (XML input application).
    • If you are using the populated data, make sure that it is correct. Edit it if necessary.
    • If you want to use a base application, select it from the Select Base Application drop-down list. The corresponding XML code is then populated in the Create Test Data area.
  5. Click the Test button.
    The Decision Module runs the data method against the XML input, and displays a message indicating the outcome of the test in the Test Result section.
  6. For a test that is successful, a View response XML link is available in the Test Result section. Click this link to view the output XML.
  7. If errors are found during the test, a View error details link is displayed in the Test Result section. Click this link to identify the errors.
