Rule Conditions

A rule evaluates conditions. Based on this evaluation, the Decision Module may execute or “fire” the rule.

A condition expression is a statement where an action takes place only if a condition is met. A simple condition expression has this form: If <field or data method> <operator> <enumeration or value> then fire this rule.

When the rule is fired, the actions defined for the rule, such as setting a reason or setting a ruleset step result, are performed.

The body of the condition is made up of three components:
  • A field or data method—The item on the left side of the expression that is being compared.
  • An operator—An action symbol in the middle of the expression that compares the items on the left and right sides.
  • A field, data method, enumeration, or value—The item on the right side of the expression to which the field or data method is being compared. An enumeration is a value that has been previously defined in the domain model as valid for the active field.
Note Comma is a special character. With the "IN" and "NOT IN" operators, do not use a comma separator inside a string.
Rule conditions are built in the Rule definition section of the Rule Properties page using the Object Model Browser and condition selection lists.
Note You must enter enumeration values manually. For more information, see the FICO Origination Manager Standalone Domain Model Specification.
