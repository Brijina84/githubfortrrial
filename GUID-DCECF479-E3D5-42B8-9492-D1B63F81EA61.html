Example: Adding a Throw Error Event

A throw handler event catches an error in a a task, call-activity, or sub-process. When the workflow or business process encocunters a throw error event, the current path of execution ends and an error is thrown. Use the throw error event with an error handler event to allow the workflow exectuion to proceed along another path. If the throw error event is not connected to an error handler event, the system throws an exception. A throw error event must have an error code associated with it.
Note The throw error is caught at processing level that is one level above. For example, if a sub-process has a throw error event attached, the error is caught at the level above a sub-process, which is at the process level. Configure the related error handler at the process level. The throw error event is at the added at the end or boundary of a task, call-activity, or sub-process. Such throw error events are used to handle custom errors. However, if a task, call-activity, or sub-process throws errors, an error handler can catch these errors. You do not add any custom throw error first.

In the following example, first create a sub-process in which you add a throw error event to Approve Application sub-process. Call this TestSubProcess. Then, create a process TestProcess and add the TestSubProcess to it. Add an error handler to catch the error and specify a human activity when the error occurs.

  1. Drag and drop the Start event.
  2. Drag and drop the Approve Application sub-process, and then connect the Start event to the sub-process.
  3. Drag and drop the Throw Error event, and then connect it to the sub-process.
  4. In the Properties pane for the Throw Error event, specify the error code.
  5. Drag and drop the End event, and then link the Throw Error event to it.
  6. Publish the TestSubProcess sub-process.
  7. Click Create next to Workflows.
  8. Drag and drop the Start event.
  9. Drag and drop the TestSubProcess sub-process, and then connect the Start event to it.
  10. Drag and drop the End event, and then connect the TestSubProcess sub-process to it.
  11. Drag and drop the Error Handler event.
  12. In the Properties pane for the Error Handler event, enter the error code that you specified for the Throw Error event and select TestSubProcess from the Node drop-down list.
  13. Drag and drop a User Activity,and connect the Error Handler event to it.
  14. Configure the User Activity event by selecting the page that must open or load for the user activity.
  15. Click Validate and correct errors, if any.
  16. Click PUBLISH.
