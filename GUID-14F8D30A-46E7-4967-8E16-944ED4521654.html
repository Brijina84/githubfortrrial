About Creating Queues

Create queues so that users or groups can work on the applications held in them.Users can have more than one queue of applications to work on, and the order in which applications in the queues are worked is defined by the administrator.

Assign users or groups to a queue so they can work on applications in that queue. You would usually assign a user or a group to a specific queue if, for example, you had a queue of sensitive applications that must only be worked by certain people.

When assigning queues to a group, you can specify the queues that members of the group can work on. Any queue can be assigned to any group in your hierarchical structure.

When creating a queue, set the properties that control the end user interface and processing (for example, select the application attributes to appear in the table for a work queue, and then configure the filter criteria for that work queue.)

As an administrator, you can specify a queue as a personal queue and select a field to be used to match the user logged in. The selected field is associated with the user name of the current user and displayed in the queue selection criteria. The fields in the drop-down list are not available for selection as filter criteria.

By default, the following fields are available when creating queues in the Application Processing Module. The default fields available are the same for both the Consumer and the Small Business templates.
