Data Method Sequences

A data method sequence is an ordered list of data methods governed by user-defined properties. When executed, it causes the included data methods to execute in a specified order on one or more specified objects.

Access data method sequences from the Item Types tab. Use the Data Method Sequence Properties page to assemble the data methods in a data method sequence.

When a decision flow includes a step with a data method sequence implementation type, the data method sequence executes when the step executes. If stored to fields, the data values derived from the execution of a data method sequence can be used later in any decision flow step where the domain model field is used.

Within a data method sequence, data methods execute in the order in which they appear in the sequence list. If a data method references an object or field that is created or modified by another data method, you must execute the data method that creates or modifies the object or field before the data method that references the newly created or modified object or field.
