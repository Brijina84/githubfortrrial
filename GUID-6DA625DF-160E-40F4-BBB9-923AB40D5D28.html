Using Product Selection Trees

Product selection trees are special decision trees that specify which product should be processed in specific circumstances. Add product selection trees in the Categories Properties page.

After you add product selection trees to a category, select a product selection tree to access the Product Selection Tree Properties page. Create variables and configure the product selection tree in the editor. The editor shows a graphical representation of the product selection tree based on various conditions and actions that can produce a decision.

A product selection tree executes in the following situations to select a product:

  • The Decision Module determines that a cross-sell operation should be performed.
  • The Decision Module receives a processing request that specifies a category to be processed, but not a product.

In both cases, the Decision Module ultimately selects a product to process based on the conditions and actions defined in the categoryâ€™s product selection tree. Then the Decision Module selects and executes a strategy for the product.

You can define multiple product selection trees and select a primary product selection tree for the category. The primary product selection tree can execute other product selection trees.

Note The Application Processing Module, can call product selection trees with the DMFunction and DMProduct fields. For detailed information, see the FICO Origination Manager Decision Module Developer's Guide.
Covered Topics
  • Example: Income Range Product Selection Tree
    You can create a product selection tree that assigns products based on income range.
  • Creating a Product Selection Tree
    Create product selection trees from the Categories Properties page. Use the Add icon in Manage Product Selection Tree to a create a product selection tree.
  • Product Selection Tree Tasks
    You can perform tasks, such as insert, copy, edit, clear, assign work on a selected tree node.
  • Inserting Splits
    Add a split to a product selection tree when you want to create a set of branches for condition nodes.
  • Inserting Levels
    All condition nodes on a single level represent a segment of the same decision variable. When you create a product selection tree, it does not contain any levels. Add a level using the Insert Level command in the Edit drop-down menu. You can also insert a level at the same time you insert a split using the Insert Split command.
  • Assigning an Expression
    The Assign Expression dialog provides the ability to author action expressions. Action expressions or statements assign a value to a property as a consequence of a series of branch nodes (conditions) evaluating to true or false. Action expressions or statements are fired from top to bottom. If there is more than one action statement that assigns a value to a property, the value assignment overwrites any assignment that was previously made.
  • Copying a Product Selection Tree
    Use the Save As icon in the Product Selection Tree Properties page to create a product selection tree from an existing product selection tree.
  • Copying and Pasting Subtrees
    A split with two or more branches is known as a subtree. You can copy a subtree from a given condition node and paste it to another condition node. You usually copy and paste subtrees when you want to repeat the same series of conditions on another branch using the same levels.
  • Reordering Branches
    Reorder the branches to rearrange their display order in the tree.
  • Creating Conditions Using Or by Merging Branches
    Merge the branches to combine condition expressions that are connected with "or". You can only merge branches in levels of enumeration or string type.
  • Creating and Mapping Variables
    Create and map variables to data fields in the object model. Variables determine the fields that condition nodes evaluate when the tree is run.
  • Exporting a Product Selection Decision Tree
    Export a product seelection decision tree as a FICO Strategy Markup Language (FSML) 2.5 file to exchange tree models with other FICO products.
  • Product Selection Tree Profiling
    Use product selection tree profiling to see how data flows through the product selection tree logic and how the actions are distributed for a given sample dataset.
