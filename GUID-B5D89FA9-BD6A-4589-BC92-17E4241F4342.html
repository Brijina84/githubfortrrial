Context Paths

The data objects and fields used to define rules in the SMW correspond to elements and attributes of the domain model schema, which also determines the structure of XML input and output. However, two slightly different styles of notation are used to refer to these items depending on whether the context is the SMW or the schema and XML.

The XPath style of notation used to describe locations in XML documents uses slashes to separate parent and child elements. In a path that points to an attribute, the attribute at the end is separated from its immediate parent element by an at (@) symbol after the slash. For example, the following path gives the location of the ProductCategory attribute of the CreditRequest element, which is a child of the Application element:

Application/CreditRequest/@ProductCategory

In the SMW path notation, elements and attributes alike are separated from each other by dots (periods), as both are regarded as fields in the SMW. There is no @ symbol to flag an attribute. For example:

Application.CreditRequest.ProductCategory
