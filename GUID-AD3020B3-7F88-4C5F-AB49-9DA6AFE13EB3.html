Product Properties

Product properties are defined in the Product Properties page. Select a product and navigate to the Product Properties page when you add or modify products.

When you select a product name in the Category Properties page, its Product Properties page appears. You can check out a product to edit product properties.

The following table describes product properties:
Property Description
Name

Name of the product.

Product name is required and must be unique within a category.

Product name can contain letters, numbers and spaces.

Code

Product code. Product code is required and must be unique within a category.

Product code can have a maximum of 50 characters and contain letters, numbers and spaces.

Product code is the value of the field, ProductCode in the Application/CreditRequest object. Communicate with your interface developer to coordinate product code values.

See Also For additional information, see the FICO Origination Manager Decision Module Developer's Guide.
Description

Description of the product.

Product description is optional.

Default decision flow

A default decision flow that is run for the product.

You can preview the decision flow properties or navigate to the decision flow.

Product Strategy

A specialized decision tree that executes a decision flow for the product.

You can preview product strategy properties or navigate to the product strategy.

Active

The start date and end date within which the product is active. Active dates are optional.

If you define active dates for a product and an XML request is received at a time outside the active date range, the request is not processed. If you choose not to set active dates, the product will be active when you promote it to production and will remain active indefinitely.

Use product threshold

Enables product severity threshold. Values are Yes and No. Default is No.

If you enable product threshold, enter an integer greater than 0 in the Product severity threshold field.

The Decision Module maintains the products's cumulative rule severity level. The Decision Module takes the following actions when you enable product threshold:
  1. Before the product’s decision flow executes its first step, the Decision Module initializes the product’s cumulative rule severity level to 0.
  2. Each time a ruleset or decision tree completes execution, the cumulative severity of all rules that executed in the ruleset is added to the product’s cumulative rule severity level.
  3. When the product’s cumulative rule severity level exceeds the product’s rule severity threshold, the decision flow is terminated.
  4. Otherwise, processing continues to the next decision flow step.
Note For more information see also Severity Settings.
Up-sell product

Select a product of higher value in the product category.

If you did not define other products in the current category, the Up-sell Product list is empty.

You can preview product properties or navigate to the product.

Cross-sell category

Select a product category that offers additional products and services.

If you did not define other categories, the Cross-sell category list is empty.

You can preview category properties or navigate to the category.

Down-sell product

Select a product of lower value in the product category.

If you did not define other products in the current category, the Down-sell Product list is empty.

You can preview product properties or navigate to the product.
