Properties Pages

Configure item type properties in the properties page. A properties page can contain editors and tool bars to modify properties.

When you select an item in the list page, the itemâ€™s properties page appears. The icons in the tool bar allow you perform certain actions on the item depending on your permissions. An editor in a properties page is the "working area" where you can build or modify item types such as decision flows.

You can perform the following actions on item types depending on the icons available and enabled in the properties page:
Action Icon Description
Check In Commits a new version of the item to the configuration repository.
Check Out Creates a working copy of the item in the private workspace.
Cancel Check Out Reverts check out of an item to undo changes.
Edit / Cancel Edit Toggles between enabling write access and disabling write access for an item.
Go to Parent Item Navigates to the parent item properties page without losing unsaved changes. You can return to the current properties page to save changes. For example, you can navigate back to a category properties page from the product properties page.
Promote Promotes a version other than the active working copy of the item.
Save Saves the changes made to the item properties.
Note If you make no changes and click the Save icon, the system does not display the standard success message that is displayed when you make any change and save it.
Save As Creates a duplicate copy of the item with a different name.
Note
You cannot create duplicate copies of the following item types:
  • Product. A product is always associated with a category.
  • Rule. A rule is always associated with a ruleset.
Test Test the conditional logic of an item with various dependent scenarios or multiple data sets. For example, you can vary the inputs to test a data method that accepts input parameters.
Verify Verifies the item and the dependent items to ensure that they are valid and generates a report.
Where Used Finds where the item is used and generates a report.
Note If you are editing a component, do not use the Refresh button of the web browser. If you do, it refreshes the connection to the server and logs you out of the SMW. Any unsaved changes are lost.
