Manage Variables

Use the Manage Variables section in a properties page to map variables to data fields available in the object model. Several item types such as score models, product selection trees, decision trees, and decision tables use variables.

When you add a variable, it is mapped to a default context and a data field in the context. You can modify the variable properties such as name, description, and type from the Variable properties page. Browse the object model to change the object and data field for a variable.

The following table describes the variable properties and their default values:
Property Description Default
Name Required. A variable name can contain letters, numbers, and underscore. variable#
Description Optional. Description can contain letters, numbers, and special characters. None
Object or reference Required. The object or reference context that the variable maps to in the object model. Application
Field or data method Required. A data field or data method in the context of the object. StrategySelectionRandomNumber
Type Required. Type can be integer, string, or real. integer
Data object reference Optional. If Yes is selected, you can select a data object reference in the object context. No

When you delete or rename a variable that is used by an item, use the verify feature to ensure there are no validation errors. You must reconfigure the item to use accurate variable names. The object model browser allows you select the fields, data methods, data objects, and data object references depending on the item you are editing and the context of your changes. Note that verification errors can occur when you delete and add a variable that is being used in a Decision Table before the private workspace is synchronized.

Note You can sort the variables when you are within the context of an item. However, the sort order is not retained when you navigate away from the properties page of the item.
