Versioning

The SMW controls user access and provides versioning of all components by implementing check out, cancel check out, and check in actions. This allows a user to safely work on components without overwriting another personâ€™s work.

Most components are versioned individually, but there are some subcomponents that are versioned with their "parent" component as shown in the following table.

Subcomponent Versioning
Top-level Component Subcomponents
Category properties Product selection tree, mapping for tree
Product strategy Mapping for initial input and results
Ruleset Rules
Decision table Mapping for table inputs and results
Decision tree Mapping for initial input and result
Score model Mapping for model inputs and results

You can also create new versions of existing products and components using check in and check out procedures in combination with the Save As command. Each time a product is checked out and then checked back in, a new version is created and saved.
