Available Roles and Permissions

When setting up the Application Processing Module, you must create users and assign specific roles to those users based on the tasks they need to perform.
Security administrators or managers (roles that are preconfigured in the Application Processing Module) manage users; system administrators manage groups and other configurations such as CRAs, business bureaus, rates, fees, products, and organizations.

System administrators can also assign users a developer role or a tester role during design and test phase. In a developer's role, you can configure plugin's registered in the system eSpaces. Access plugin's from Settings task menu. In a tester's role, you can access the Application Processing Module user interface pages that allow testing queues.

The Application Processing Module comes preconfigured with two sets of roles: roles that are part of the "base" application (OM APM) and roles used in template applications (for example, OM APM Consumer or OM APM Small Business). The template roles control how a technician can define their company's roles and expose those roles in the administrative pages where administrators or managers can assign roles to users and groups.

The following table lists the available roles that exist in the system upon initial use, describes the roles, and provides an example of the type of user usually assigned to each role. Administrators can choose any of these available roles and mix or match them while assigning them to users or groups.

Note "Manage" in the role names below includes create, edit, view, and delete permissions. For example, a user with the ManageRate role can add, duplicate, modify, activate, deactivate, and delete rates in the system.
See Also For more information on the base and template roles and how you can use FICO Application Studio Designer to extend available roles within the consumer and the small business templates, see the FICO Origination Manager Application Processing Module Developer's Guide and the FICO Origination Manager Integration Guide.
