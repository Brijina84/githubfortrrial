Comparison Expressions in Conditional Statements

You can use simple and complex comparison expressions in conditional statements. A simple comparison expression is syntactically valid if it is of the form: 
<operand1> <conditional_operator> <operand2>
Where:
  • <operand1> is a data method expression identifier whose type is valid for the specified <conditional_operator>.
  • <operator> is any of the comparison operators that is valid for the data type of <operand1>. (See the following table.)
  • <operand2> is the operand to which <operand1> will be compared and may be any expression identifier whose data type matches that of <operand1>.
The following table contains an example of a comparison statement: 
TotalMonthlyPaymentsAmount >= 25000
Comparison Operators
Operator Return Value Valid Data Types
> Returns true if operand 1 is strictly greater than operand 2 date, duration, numeric, string, time, timestamp
>= Returns true if operand 1 is greater than or equal to operand 2 date, duration, numeric, string, time, timestamp
< Returns true if operand 1 is strictly less than operand 2 date, duration, numeric, string, time, timestamp
<= Returns true if operand 1 is less than or equal to operand 2 date, duration, numeric, string, time, timestamp
= Returns true if operand 1 is equivalent to operand 2 boolean, date, duration, numeric, object, string, time, timestamp
<> Returns true if operand 1 is not equivalent to operand 2 boolean, date, duration, numeric, object, string, time, timestamp
Note Objects can only be compared to null in comparison statements.
Complex comparison expressions are constructed by combining multiple simple expressions, using multiple operators, and adding parentheses to specify precedence, such as in the following example: 
((a > b) and (b <= c)) or (b > d)
