Assignment Statements

A simple assignment statement contains at least one expression containing the := assignment operator. It may contain additional conditional expressions. In an assignment statement, the operation on the right of the operator produces a result that is assigned to the value on the left.
An assignment statement has the following basic syntax: 
statement_result := operation;

where both sides of the expression have the same data type.

The left side (statement_result) of the expression can be any of the following:
  • The keyword result, which represents the data method result value (such as “result := 3;”).
  • A local that has been defined for the data method.
  • The name of a field or field identifier that is valid for the active context (object.field, input.field, local.field, datamethod.field, or result.field).
Note Field names may take the form Object.FieldName, where Object is a domain model object; or a local, input, data method, or function; or the keyword result that is an object type.
The right side (operation) of the expression can be any of the following, provided that it has the same data type or produces a result or return value of the same data type as the left side (statement_result) and is valid for the active context:
  • A constant value, such as “abc” or 5
  • An operation
  • The null keyword
  • An identifier—a unique name of an object, input, local, data method, function call, internal service, field, value, or comparison expression
Numeric and string operations used in the right side of an assignment statement can be simple or complex, and can have a number of syntactic forms. For example, a valid simple numeric operation could be any of the following:
  • An arithmetic operation of the form <operand1> <arithmetic_operator> <operand2> where:
    • <operand1> is a numeric identifier, such as a local, input, field, value, data method, or function, or the keyword result
    • <arithmetic_operator> is a valid arithmetic operator (+, -, *, or /)
    • <operand2> is another numeric identifier
  • A function call with a numeric return value
  • A data method call with a numeric result value

You can create more complex operations with multiple operators and operands.

A valid simple string operation could be any of the following:
  • String operation of the form <operand1> + <operand2>
  • A function call with a string return value
  • A data method call with a string result value
The default expression that appears in the Expression Editor is an example of a simple assignment statement: 
result := null;

This statement assigns an initial value to result (initializes it) by assigning it the value on the right, null.
