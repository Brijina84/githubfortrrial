Transaction Data and Decisioning

This section provides information on various Decision Module decision flow components and their relation to the Decision Module domain model and transaction data.

Covered Topics
  • Context Paths
    The data objects and fields used to define rules in the SMW correspond to elements and attributes of the domain model schema, which also determines the structure of XML input and output. However, two slightly different styles of notation are used to refer to these items depending on whether the context is the SMW or the schema and XML.
  • Data Object References
    Decision flow components such as data methods, data object references, rules, and mappings need a context, or specified set of objects, on which to operate. You specify a context by choosing the desired object from the Data Object Browser in the SMW.
  • Data Methods
    Data methods are functions that perform computational tasks, such as arithmetic operations, string manipulations, date and time field manipulations, and object creation and handling. They can be used to generate the data needed in business policies (rules).
  • Data Method History Output
    For testing and debugging purposes, the Decision Module can return detailed information on data methods executed during decisioning.

  • Processing History Output
    The ProcessingHistory output node contains information on the product strategy and decision flow components that were executed during transaction processing.
  • Using User-defined Fields
    The UserDefinedField object enables you to create user-defined fields to hold data dynamically created by the Decision Module for use in processing the current request.
  • User-defined Fields Created or Populated by the Decision Module
    To use a user-defined field as a global variable, a data method can scan a request for some trait and then set a user-defined field to indicate presence of the trait. For example, a user-defined field might be used to flag whether the applicant is an existing customer.
  • Resubmitting Transactions
    To indicate that a transaction is a resubmission, you must enter a value of “true” for the Application/@ResubmissionFlag.
  • Updating the Data Repository with Final Decision Data
    At runtime, the Process Server sends XML transaction data to the DSS. The FinalApplicationDecision is the final application decision of the calling system (as opposed to the application decision recommended by the Decision Module).
