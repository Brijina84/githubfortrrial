Rules

A rule is a combination of one or more conditions with a resulting decision reason and severity level that represent your organization’s decisions and policies. Rules are always associated with rulesets and are the basic unit of rule processing in decision flow steps that have a ruleset implementation type. A ruleset can have one or more rules.
See Also An understanding of the domain model structure and conventions is essential when creating and managing rules. For information on domain model concepts, see Defining and Evaluating Business Strategies.
Covered Topics
  • Rule Properties
    The Rule Properties page displays the properties associated with a rule, such as its context, decision reason, severity level, and the conditions under which the rule returns an outcome.
  • Rule Context
    A rule’s context is the full path of the domain model item on which it is defined.The context path is displayed as a sequence of object names, separated by periods, with the first object always being the Application object.
  • Rule Scope
    A rule may have one condition or many. A rule’s logic is constructed such that it can fire if any condition is true or all conditions are true, depending on the setting you choose.
  • Rule Conditions
    A rule evaluates conditions. Based on this evaluation, the Decision Module may execute or “fire” the rule.
  • Managing Rules
    Use the options on the properties page for a ruleset to add and remove rules. Use the options on the properties page for a rule to define its outcome settings, scope, and conditions.
